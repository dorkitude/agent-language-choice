# Pilot Results: 2026-07-08

Initial smoke test for the codegen benchmark harness.

## Scope

- Provider: `pi` with Fireworks
- Models: `glm-5p2`, `kimi-k2p7-code`
- Tasks: `kv_patch`; later same-day smoke extension for `ledger_debug`
- Languages: Go, TypeScript; later same-day smoke extension for Java/PHP
- Scoring: black-box stdin/stdout tests through generated `./run.sh`
- Newly installed toolchains: PHP 8.5.8, OpenJDK/Javac 26.0.1

## Notes

All runs passed, so this pilot is not evidence of a correctness gap.
However, TypeScript took longer for both models on the same task:

- GLM 5.2: TypeScript was about 2.36x Go wall-clock agent time.
- Kimi 2.7 Code: TypeScript was about 4.06x Go wall-clock agent time.

This is consistent with the project hypothesis that language/tooling
differences may show up in repair/search cost before they show up in pass/fail
on easy tasks.

The Java/PHP extension verifies that PHP and OpenJDK are installed and usable
through the harness. The `ledger_debug` rows verify that seeded pre-existing
repos can be materialized, repaired, and scored through the same black-box
contract.

The next pilot should add harder tasks (`route_params`, `window_counts`) across
all six languages and record iteration/error classes explicitly.
