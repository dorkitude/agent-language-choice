## Solution files

**`solution.ts`** — the implementation:
- Reads stdin synchronously via `fs.readFileSync(0, 'utf8')`
- Parses each non-empty line as JSON, validating required field types
- **Stable sort** by `ts` ascending, using an explicit input-index as the tiebreaker (deterministic regardless of engine sort stability)
- Applies `set` (stores string value) / `delete` (removes key) in order
- Sorts output keys **bytewise** via `Buffer.from(k,'utf8').compare(...)` (true UTF-8 byte order = code-point order, correct even for non-ASCII)
- Prints `key=value` lines, or nothing if the map is empty

**`node-globals.d.ts`** — minimal ambient declarations for `fs`, `process`, and `Buffer` so `tsc` compiles cleanly *without* `@types/node` (which isn't installed).

**`run.sh`** — POSIX `#!/bin/sh` script that `cd`s to its own dir, compiles with `tsc` (no `--moduleResolution node` since TS 6 deprecated it), then runs `node solution.js`.

## Verification
All 3 task tests pass, plus edge cases: empty input, no trailing newline, bytewise ordering with mixed case (`Banana` < `apple`), delete-missing no-op, `=` in values, and same-ts set-then-delete. Only the Node standard library is used — no npm packages or `tsx`. ✨
