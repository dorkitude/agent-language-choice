**What's here:**
- `Main.java` — a single-file server using only the JDK stdlib (`com.sun.net.httpserver.HttpServer`), with a hand-rolled JSON parser/serializer (no third-party deps 🧙).
- `run.sh` — compiles then `exec java Main` in the foreground, listening on `127.0.0.1:$PORT`.

**All 5 endpoints tested green against the spec's sample payloads:**
- `GET /health` → `{"ok":true}` ✅
- `POST /v1/dice/stats` → correct stats + HTTP 400 on invalid expressions ✅
- `POST /v1/checks/ability` → total/success/margin ✅
- `POST /v1/encounters/adjusted-xp` → `base_xp:850, multiplier:2, adjusted_xp:1700, difficulty:"deadly"` ✅
- `POST /v1/initiative/order` → sorted by score desc, dex desc, name asc ✅

A few deliberate choices:
- Integers are emitted without decimals (e.g. `multiplier:2`, `average:10`) via a `num()` helper that drops `.0`, matching the expected JSON.
- Difficulty picks the highest threshold reached, falling through to `trivial`.
- Bad bodies / unknown CRs / unsupported levels return HTTP 400.

The server was **not** left running — it's stopped and ready for the benchmark harness to launch via `./run.sh`. 🏁
