I implemented the D&D REST API in Go stdlib and verified it builds and runs.

**Files created/modified:**
- `main.go` — the full server
- `run.sh` — already existed, still works (`go run .`)

**What it does:**
- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` — parses `NdS+M`, returns stats including average as `(min+max)/2`
- `POST /v1/checks/ability` — total, success, margin
- `POST /v1/encounters/adjusted-xp` — supports CRs 0–5, level-3 thresholds, multipliers by monster count
- `POST /v1/initiative/order` — sort by score desc, dex desc, name asc

**Verified outputs** match the spec examples exactly:
- Dice: `{"dice_count":2,"sides":6,"modifier":3,"min":5,"max":15,"average":10}`
- Ability: `{"total":14,"success":false,"margin":-1}`
- XP: `{"base_xp":850,"monster_count":3,"multiplier":2,"adjusted_xp":1700,"difficulty":"deadly","thresholds":{...}}`
- Initiative: `{"order":[{"name":"rogue","score":17},{"name":"ogre","score":15}]}`

No third-party packages were added. The server listens on `127.0.0.1` using the `PORT` env var (defaults to 8080).
