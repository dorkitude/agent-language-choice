## Summary

Built a D&D REST engine in **Java 26 + stdlib only** (`com.sun.net.httpserver.HttpServer`) — no external deps, hand-rolled JSON parser/serializer.

**Files:**
- `Main.java` — single-file implementation
- `run.sh` — `javac Main.java && java Main` (foreground, reads `PORT`, binds `127.0.0.1`)

**Endpoints verified byte-for-byte against the spec:**
| Endpoint | Result |
|---|---|
| `GET /health` | `{"ok":true}` ✓ |
| `POST /v1/dice/stats` `2d6+3` | `{"dice_count":2,"sides":6,"modifier":3,"min":5,"max":15,"average":10}` ✓ |
| `POST /v1/checks/ability` | `{"total":14,"success":false,"margin":-1}` ✓ |
| `POST /v1/encounters/adjusted-xp` | `{"base_xp":850,"monster_count":3,"multiplier":2,"adjusted_xp":1700,"difficulty":"deadly","thresholds":{"easy":300,"medium":600,"hard":900,"deadly":1600}}` ✓ |
| `POST /v1/initiative/order` | `{"order":[{"name":"rogue","score":17},{"name":"ogre","score":15}]}` ✓ |

**Edge cases checked:**
- Dice `1d20` → `average:10.5` (exact math, halves rendered as decimals)
- Invalid dice (`0d6`), unknown CR, malformed JSON → `400`
- Wrong method → `405`, unknown path → `404`
- 2-monster multiplier → `1.5`, fractional `adjusted_xp:52.5` (not rounded — spec says literal multiply)
- Initiative tie-breaks: score desc → dex desc → name asc ✓
- Difficulty `trivial` when below easy threshold ✓

`run.sh` is ready to go — it compiles and starts the server in the foreground. 🗡️
