Done! 🎲✨

Built the D&D REST engine in pure Python 3.14 stdlib (`http.server` + `json`), Chef. All five endpoints verified against the spec examples and matching byte-for-byte:

- 🩺 `GET /health` → `{"ok": true}`
- 🎲 `POST /v1/dice/stats` — regex-parsed `<count>d<sides>[±mod]`, positive-count/sides enforced, HTTP 400 on invalid
- ⚔️ `POST /v1/checks/ability` — total/success/margin
- 🐉 `POST /v1/encounters/adjusted-xp` — CR→XP table, count multipliers, summed level-3 thresholds, highest-reached difficulty
- 🏹 `POST /v1/initiative/order` — sorted by score↓, dex↓, name↑

A couple of deliberate touches:
- **Integer-clean numbers**: `average`, `multiplier`, and `adjusted_xp` collapse to ints when whole (so `10` not `10.0`, `2` not `2.0`) to match the expected JSON.
- **Bool guard**: JSON `true`/`false` are rejected where integers are required (Python treats `bool` as `int`, so this matters).
- `ThreadingHTTPServer` on `127.0.0.1:$PORT` (defaults to 8000), `run.sh` `exec`s it in the foreground.

The server is **not** running — I tested it on a throwaway port and killed it. `./run.sh` is ready. 🚀
