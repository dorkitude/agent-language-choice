I created a minimal, deterministic Python 3.14.6 stdlib server in `server.py` and wired it up via `run.sh`.

**Files:**
- `server.py` — implements all five endpoints using `http.server` and `json`
- `run.sh` — `exec python3 server.py` (foreground, respects `PORT`)

**Verified outputs:**
- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` → `{"dice_count": 2, "sides": 6, "modifier": 3, "min": 5, "max": 15, "average": 10}`
- `POST /v1/checks/ability` → `{"total": 14, "success": false, "margin": -1}`
- `POST /v1/encounters/adjusted-xp` → `{"base_xp": 850, "monster_count": 3, "multiplier": 2, "adjusted_xp": 1700, "difficulty": "deadly", "thresholds": {...}}`
- `POST /v1/initiative/order` → `{"order": [{"name": "rogue", "score": 17}, {"name": "ogre", "score": 15}]}`

Invalid dice expressions correctly return HTTP 400. The server is not running now — `./run.sh` is ready to start it on `127.0.0.1:$PORT` when you are.
