## Summary

Built the D&D REST API as Django views inside the seeded minimal project (`dndsite/`).

**Files touched:**

- `run.sh` — already correct, left as-is (foreground `runserver --noreload` on `127.0.0.1:$PORT`)

**Endpoints implemented:**
| Endpoint | Behavior |
|---|---|
| `GET /health` | `{"ok": true}` |
| `POST /v1/dice/stats` | Regex-parsed `<count>d<sides>[±<mod>]`; validates count/sides > 0; returns `dice_count, sides, modifier, min, max, average` (int when integral, float like `3.5` otherwise) |
| `POST /v1/checks/ability` | `total = roll+modifier`, `success = total >= dc`, `margin = total - dc` |
| `POST /v1/encounters/adjusted-xp` | CR→XP table, monster-count multipliers (1.5/2.5 stay fractional), summed per-level thresholds, difficulty = highest reached (`trivial`→`deadly`) |
| `POST /v1/initiative/order` | `score = roll+dex`, sorted by score desc → dex desc → name asc |

**Verified against the spec** — every sample response matches byte-for-byte, plus tie-breaking, fractional multipliers (`1.5`, `2.5`), the `trivial` band, negative modifiers, and 400s for invalid expressions, bad JSON, unsupported CRs, and unsupported levels. Django `check` passes clean on Python 3.14.6 / Django 6.0.7. ✨
