I added the three stateful combat endpoints to `vite.config.ts` while preserving every existing core/character-rule endpoint:

- `POST /v1/combat/sessions` — creates a session with client-supplied ID, computes initiative scores, sorts by score → dex → name, and starts at `round=1`, `turn_index=0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition with positive `duration_rounds` to an existing combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances `turn_index`, wraps to the next round, and decrements/removes conditions on the newly active combatant.

State is stored in memory in a `Map`. Unknown session IDs return **404**; malformed input returns **400**. All success responses are JSON.

I verified with quick local curls that:
- session creation matches the spec example exactly
- conditions attach correctly
- advancing decrements active-combatant conditions and wraps rounds
- `404` is returned for missing sessions
- all existing endpoints (e.g., `/v1/initiative/order`) still work
