Fixed! 🎯

**Root cause:** `formatCombatConditions` in `vite.config.ts` was only including combatants that still had active conditions. When a condition expired, the combatant's key disappeared from the `conditions` object, causing the `combat-advance-expired-removed` test to fail because it expected `"fighter"` to still be present (with an empty array).

**Fix:** Always include every combatant in the `conditions` response, even if their condition list is empty.

```typescript
// vite.config.ts
function formatCombatConditions(session: CombatSession): any {
  const conditions: Record<string, ...> = {};
  for (const c of session.combatants) {
    conditions[c.name] = c.conditions.map(...); // no more empty-array skip
  }
  return conditions;
}
```

Typecheck passes, `run.sh` is preserved, and I haven't started the server. 🚀
