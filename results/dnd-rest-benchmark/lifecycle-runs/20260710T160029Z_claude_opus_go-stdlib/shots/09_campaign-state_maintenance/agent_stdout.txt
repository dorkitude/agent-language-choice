All endpoints work correctly and match the spec exactly. Prior-stage behavior is preserved (health, dup=409, 404, 400 all correct). ✨

`run.sh` already runs the server in the foreground via `go run .` on `127.0.0.1:$PORT`, so it's ready as-is.

Summary of what I added, Chef 🎲:

- **`POST /v1/campaigns`** — create campaign (409 on duplicate id, 400 on missing fields)
- **`POST /v1/campaigns/{id}/characters`** — add character (404 unknown campaign, 409 dup char id)
- **`POST /v1/campaigns/{id}/events`** — add log event, returns `{id, kind}`, bumps `log_count`
- **`GET /v1/campaigns/{id}/state`** — full state with `characters` array and `log_count`

Campaign state is stored in a mutex-guarded map and cleared on `POST /v1/storage/reset` alongside the other in-process stores, consistent with the existing SQLite-file convention in this codebase. No third-party packages, stdlib only. ✅
