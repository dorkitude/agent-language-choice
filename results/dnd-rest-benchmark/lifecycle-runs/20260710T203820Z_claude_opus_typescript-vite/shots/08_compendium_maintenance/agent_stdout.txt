Done! 🎲 Stage 5 (Monster & Item Compendium) is implemented and verified.

## What I added 🐉

All within the existing Vite dev-server middleware (`vite.config.ts`) — no framework swap, versions untouched:

**Schema** — two new SQLite tables (`monsters`, `items`) created via `CREATE TABLE IF NOT EXISTS`, so the existing `game.db` upgrades cleanly. `resetStorage` now drops them too.

**Endpoints**
- `POST /v1/compendium/monsters` — create; response omits `tags` per spec 📜
- `GET /v1/compendium/monsters/:slug` — read; includes `tags`
- `POST /v1/compendium/items` — create ⚗️
- `GET /v1/compendium/items/:slug` — read

**Status codes** ✅
- `400` malformed input (bad slug/name/cr/ints/tags)
- `404` unknown record
- `409` duplicate slug (reusing the existing `Conflict` class)

**Verified live**: create/read round-trips, `409` on dup goblin, `404` on missing, `400` on bad body, item flow, and a prior-stage endpoint (`dice/stats`) all pass. Tags stored as JSON, defaulting to `[]` when omitted.

`./run.sh` is unchanged and ready — foreground Vite on `127.0.0.1:$PORT`. Nothing left running. 🧹
