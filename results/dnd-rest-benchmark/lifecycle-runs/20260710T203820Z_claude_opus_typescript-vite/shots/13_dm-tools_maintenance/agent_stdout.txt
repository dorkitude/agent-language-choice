## What I added

I implemented three DM-facing handlers in `vite.config.ts` (the existing Vite plugin-middleware API — no framework swap), reusing the existing compendium/campaign-state stores and the core adjusted-XP math:

- **`POST /v1/dm/encounter-builder`** 🐉 — looks up each monster's CR from the SQLite compendium, sums base XP (`CR_XP`), applies the shared `encounterMultiplier`, and rates difficulty against the party's summed `LEVEL_THRESHOLDS`. `3× goblin (CR 1/4)` → `base_xp 150`, `adjusted_xp 300`, `easy` → `"safe warm-up"`.
- **`POST /v1/dm/loot-parcel`** 💰 — deterministic tier table; tier 1 → `75 gp` + `healing-potion ×2`.
- **`POST /v1/dm/session-recap`** 📜 — pulls the latest campaign event as the summary and derives an open thread from the trailing "…the *goblin trail*." clause → `"Resolve goblin trail ambush"`. Unknown campaign → `404`.
