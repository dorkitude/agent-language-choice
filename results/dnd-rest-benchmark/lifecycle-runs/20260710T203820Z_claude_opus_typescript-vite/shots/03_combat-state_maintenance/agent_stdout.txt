- **`POST /v1/combat/sessions`** — creates an in-memory session with initiative sorted by `score` desc → `dex` desc → `name` asc, starting at `round=1`, `turn_index=0`. Rejects duplicate IDs (`400`).
- **`POST /v1/combat/sessions/{id}/conditions`** — attaches a condition to a named combatant; validates positive-integer duration and known target; `404` for unknown session.
- **`POST /v1/combat/sessions/{id}/advance`** — advances `turn_index`, bumps `round` on wrap, decrements conditions **only** on the newly-active combatant and drops any that hit `0`.

- Path-param routes handled via regex before the exact-match `ROUTES` table, so the original router stays intact.
- Conditions live in a `Map<name, Condition[]>` per session; empty entries are pruned so the `conditions` object stays clean.
