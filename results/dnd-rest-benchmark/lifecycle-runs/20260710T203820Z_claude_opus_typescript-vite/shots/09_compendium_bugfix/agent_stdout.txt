The bug was in the middleware dispatch at `vite.config.ts:701` — only `POST /v1/auth/register` was hardcoded to return `201`, while every other POST (including the new compendium create routes) fell back to `200`. The evaluator wanted `201 Created` for the new monster/item creates.

**The fix** 🔧: replaced the single-route check with a `CREATED_ROUTES` set covering register + the two compendium creates:

```ts
const CREATED_ROUTES = new Set([
  'POST /v1/auth/register',
  'POST /v1/compendium/monsters',
  'POST /v1/compendium/items',
]);
const successStatus = CREATED_ROUTES.has(route) ? 201 : 200;
```
