I added the **stateful combat** endpoints to `main.go` while keeping every existing core & character endpoint intact:

- `POST /v1/combat/sessions` — creates an initiative-sorted session
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition to a combatant
- `POST /v1/combat/sessions/{id}/advance` — advances turn order, wraps rounds, and ticks down conditions on the newly active combatant only

All state lives in memory under a `sync.RWMutex`-protected map. Invalid input returns `400`, unknown session IDs return `404`, and success responses are JSON.

`./run.sh` already starts the server in the foreground on `127.0.0.1:$PORT` with `go run .`, so no changes were needed there.

Verified it builds cleanly with `go build .` ✅. Let the evaluator roll initiative!
