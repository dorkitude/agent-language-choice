Fixed! 🔧

The combat advance handler was filtering out combatants whose condition list had become empty, so `conditions` showed `{}` instead of keeping the `"fighter"` key with an empty array.

**Change in `main.go`:**

```go
respConds := make(map[string][]conditionEntry)
for name, conds := range s.conditions {
    copyConds := make([]conditionEntry, len(conds))
    copy(copyConds, conds)
    respConds[name] = copyConds
}
```

Now every combatant that ever had a condition remains in the response with an `[]` slice once all conditions expire.

Verified with a local httptest scenario matching the failing evaluator flow — the response now includes:

```json
{"id":"enc-1","round":2,"turn_index":2,"active":{"name":"fighter","score":14},"conditions":{"fighter":[]}}
```

Project builds cleanly and `./run.sh` is unchanged. Ready to rock, Chef! 🎲✨
