I added the stateful combat endpoints while preserving every existing core/character endpoint. Here is what is now in place:

### New files
- `app/lib/combat.ts` — in-memory combat store and business logic (initiative sort, conditions, turn advancement).
- `app/v1/combat/sessions/route.ts` — `POST /v1/combat/sessions` to create a session.
- `app/v1/combat/sessions/[id]/conditions/route.ts` — `POST /v1/combat/sessions/{id}/conditions`.
- `app/v1/combat/sessions/[id]/advance/route.ts` — `POST /v1/combat/sessions/{id}/advance`.

### Behavior preserved
- Initiative scoring is `roll + dex`, sorted by score desc → dex desc → name asc.
- New sessions start at `round = 1`, `turn_index = 0`.
- Conditions are attached to the named combatant.
- Advance increments `turn_index`, wraps to `0` and increments `round`, then decrements conditions on the newly active combatant and removes expired ones.
- Invalid input returns `400`; unknown session IDs return `404`.
