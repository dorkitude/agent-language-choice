Fixed, Chef! 🎲✨

**Root cause:** In `advanceTurn`, when a condition expired we `delete`d the combatant’s entry from the conditions map. The evaluator expected the active combatant’s key to remain in the response, even if its condition list was empty.

**Fix:** `app/lib/combat.ts` now keeps an empty array for the active combatant when it had conditions that all expired, instead of deleting the key.

```typescript
// before
if (remaining.length > 0) {
  session.conditions[activeName] = remaining;
} else {
  delete session.conditions[activeName];
}

// after
if (activeConditions.length > 0) {
  session.conditions[activeName] = remaining;
} else {
  delete session.conditions[activeName];
}
```

So after the condition on `fighter` expires, the response now returns:

```json
{
  "id": "enc-1",
  "round": 2,
  "turn_index": 2,
  "active": { "name": "fighter", "score": 14 },
  "conditions": { "fighter": [] }
}
```
