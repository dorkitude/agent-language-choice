Done, Chef! ✅ Added the Stateful Combat stage to the Rust stdlib D&D REST server.

What changed in `src/main.rs`:
- Added in-memory session storage via `LazyLock<Mutex<HashMap<…>>>`.
- Added `Session`, `Combatant`, and `Condition` structs.
- Added three new endpoints:
  - `POST /v1/combat/sessions` — creates a session with initiative order.
  - `POST /v1/combat/sessions/{id}/conditions` — attaches a condition to a combatant.
  - `POST /v1/combat/sessions/{id}/advance` — advances turn, wraps rounds, and decrements active combatant conditions.
- Preserved all existing `/health`, `/v1/dice/*`, `/v1/checks/*`, `/v1/encounters/*`, `/v1/initiative/*`, and `/v1/characters/*` endpoints.
- No new Cargo dependencies; still using only `std`.

`./run.sh` is ready and unchanged:
```bash
#!/usr/bin/env bash
set -euo pipefail
rustc --edition=2024 src/main.rs -o dndrest
./dndrest
```

I compiled successfully and ran a quick smoke test against the example requests — create session, add condition, and advance all returned the expected JSON shapes. Server is not running now; run `./run.sh` to start it when you're ready. 🎲⚔️
