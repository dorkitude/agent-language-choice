Implemented combat state support in `server.py`:

- `POST /v1/combat/sessions`
- `POST /v1/combat/sessions/{id}/conditions`
- `POST /v1/combat/sessions/{id}/advance`

State is in memory, initiative ordering follows the specified tie-breaks, conditions decrement only when their target becomes active, and unknown sessions return `404`.

Verification run without starting the server:
- `python3 -m py_compile server.py`
- Direct combat-state checks for creation, condition add, turn advance, round wrap, and condition expiry
- Confirmed `run.sh` remains executable and starts `python3 server.py` in the foreground.
