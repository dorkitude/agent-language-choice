I added the three new combat endpoints while preserving all existing behavior:

- `POST /v1/combat/sessions` — creates a session with client-supplied ID, computes initiative order (`score = roll + dex`, ties broken by dex desc then name asc), starts at `round=1`, `turn_index=0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition with positive duration to a named combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances the turn, wraps the round, and decrements/removes conditions on the newly active combatant only.
