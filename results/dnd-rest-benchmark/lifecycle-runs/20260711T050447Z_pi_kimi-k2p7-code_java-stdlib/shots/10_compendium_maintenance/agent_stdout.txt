All set, Chef! ✅

I added the SQLite-backed compendium APIs on top of the existing D&D REST server:

- `POST /v1/compendium/monsters` → creates a monster with tags (returns 201, omits tags as per spec)
- `GET /v1/compendium/monsters/{slug}` → returns full monster including tags (200 / 404)
- `POST /v1/compendium/items` → creates an item (201)
- `GET /v1/compendium/items/{slug}` → returns item (200 / 404)
- Duplicate slug returns 409; malformed input returns 400

Implementation details:
- Added `monsters`, `monster_tags`, and `items` tables in both `init()` and `reset()`.
- Added `SQLiteStore` helper methods for existence checks, inserts, and JSON-loaded reads.
- Added `MonstersHandler` and `ItemsHandler` using the existing path-pattern style.
- Preserved every existing endpoint and behavior.

`./run.sh` is ready and compiles cleanly. The server will listen on `127.0.0.1:$PORT` and run in the foreground via `exec java Main`.
