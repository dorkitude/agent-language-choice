Chef, the SQLite storage stage is wired up and ready! 🎲🗄️

What changed in `Main.java`:
- Added `SQLiteStore` that uses `sqlite3` via standard `ProcessBuilder` to persist durable data in `game.db`.
- Moved combat sessions and user accounts from in-memory maps to SQLite tables.
- Preserved every previous endpoint exactly (health, dice, checks, encounters, initiative, character stats, combat, auth).
- Added new endpoints:
  - `GET /v1/storage/status` → `{"driver":"sqlite","schema_version":1,"initialized":true}`
  - `POST /v1/storage/reset` → clears data and recreates the schema, returns `{"ok":true,"schema_version":1}`

`./run.sh` is unchanged and ready to compile & run in the foreground. No server is running now.
