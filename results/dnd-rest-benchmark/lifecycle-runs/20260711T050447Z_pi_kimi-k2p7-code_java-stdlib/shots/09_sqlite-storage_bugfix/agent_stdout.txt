**Root cause:** In `CombatSessionsHandler.advanceTurn`, when a condition’s `remaining_rounds` hit `0`, the code was removing the entire combatant key from the `conditions` map if the list became empty. The `combat-advance-expired-removed` test expects the active combatant’s key to remain with an empty array (`"fighter": []`), so the map was coming back as `{}` instead.

**Fix:** I removed the `if (activeConditions.isEmpty()) session.conditions.remove(activeName);` cleanup in `Main.java`. Expired conditions are still pruned from the list, but the combatant key stays present with an empty list so the response matches the evaluator’s contract.
