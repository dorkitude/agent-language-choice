Done, Chef! 🎲✨

I added three new deterministic PHB-rule endpoints to the existing Java `HttpServer` app, keeping every prior endpoint intact.

**What changed:**
- `Main.java`
  - Added context registrations:
    - `/v1/phb/spell-slots`
    - `/v1/phb/rests/long`
    - `/v1/phb/equipment-load`
  - Added three new minimal `HttpHandler` classes before the existing `Json` helper.

**Implementation notes:**
- Spell slots only supports the specified case: `wizard` level 5 → `{"1":4,"2":3,"3":2}`.
- Long rest: restores HP to max, restores `max(1, floor(level/2))` spent hit dice (min 0), and reduces exhaustion by 1 (min 0).
- Equipment load: `capacity = strength * 15`, `encumbered = weight > capacity`.

**Verification (curl smoke tests):**
```bash
# /v1/phb/spell-slots
{"class":"wizard","level":5,"slots":{"1":4,"2":3,"3":2}}

# /v1/phb/rests/long
{"hp_current":35,"hit_dice_spent":1,"exhaustion_level":0}

# /v1/phb/equipment-load
{"capacity":180,"weight":181,"encumbered":true}
```

Existing endpoints like `/health` and `/v1/dice/stats` still work. `./run.sh` is unchanged and ready to go.
