Done, Chef! 🎲 Here's what I built for the **core D&D REST engine** in Python/Flask 3.1.3:

## Files

**`app.py`** — Flask routes for all four endpoints:
- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` — regex-parsed `<count>d<sides>[±<mod>]`, validates positivity, returns `dice_count/sides/modifier/min/max/average`; true mean `(min+max)/2` (collapses whole floats → int to match the sample); invalid → 400
- `POST /v1/checks/ability` — `total=roll+modifier`, `success=total>=dc`, `margin=total-dc`
- `POST /v1/encounters/adjusted-xp` — CR→XP table, monster-count multipliers, summed party thresholds (level-3), picks highest difficulty reached; unknown CR → 400
- `POST /v1/initiative/order` — `score=roll+dex`, sorts by score desc → dex desc → name asc

**`run.sh`** — cds to its own dir, sets `PYTHONPATH=.deps`, auto-installs Flask (uv, falling back to pip) only if missing, then `exec python3 app.py` in the foreground on `127.0.0.1:$PORT`.

**`requirements.txt`** — `Flask==3.1.3` (unchanged pin). `.deps/` pre-populated so the server starts immediately.
