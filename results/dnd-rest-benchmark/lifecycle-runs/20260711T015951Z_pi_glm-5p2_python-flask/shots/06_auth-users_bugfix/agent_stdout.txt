## Summary

**Root cause:** The `POST /v1/auth/register` endpoint returned the default HTTP 200 status, but the benchmark's `auth-register-user` test expects **HTTP 201 Created** for successful user registration.

**Fix:** A one-line change — added `, 201` to the successful register response:

```python
return jsonify(username=username, role=role), 201
```

Everything else (password hashing via `werkzeug.security`, 409 for duplicates, 401 for bad credentials, 400 for malformed input, and all prior-stage endpoints) was already correct and remains untouched. The login endpoint correctly returns 200 with `session-<username>` token.

`./run.sh` is ready to go — it starts the Flask server in the foreground on `127.0.0.1:$PORT`. 🐍⚔️
