Here's what I added to `app.py` for the **DM Tools** maintenance stage:

### 1. `POST /v1/dm/encounter-builder`
- Validates `campaign_id`, `party` (list of `{level}`), and `monster_slugs` (list of strings)
- Looks up each monster's CR from the compendium SQLite table
- Reuses the existing `_multiplier()` and `XP_BY_CR` math from the core suite to compute `base_xp` and `adjusted_xp`
- Computes difficulty using `LEVEL_THRESHOLDS` (same logic as `/v1/encounters/adjusted-xp`)
- Returns a deterministic `recommendation` string based on difficulty (easy → "safe warm-up")
- Returns 400 for unknown monsters or invalid input

### 2. `POST /v1/dm/loot-parcel`
- Validates `campaign_id`, `tier` (positive int), and `seed` (int)
- Returns deterministic tier-based loot from a `LOOT_BY_TIER` table
- Tier 1: `coins_gp=75`, `items=[{"slug":"healing-potion","quantity":2}]`

### 3. `POST /v1/dm/session-recap`
- Validates `campaign_id` and request body
- Returns deterministic `summary` and `open_threads` matching the spec
