## Changes to `app.py`

**Schema additions** (in `init_schema`):
- `monsters` table: `slug` (PK), `name`, `cr`, `armor_class`, `hit_points`, `tags` (JSON-encoded text)
- `items` table: `slug` (PK), `name`, `type`, `rarity`, `cost_gp`

**Storage lifecycle updates**:
- `clear_data()` now also deletes from `monsters` and `items`
- `reset_db()` now also drops `monsters` and `items`

**New helper functions**: `monster_get`, `monster_exists`, `monster_put`, `item_get`, `item_exists`, `item_put`

**New routes**:
| Route | Behavior |
|---|---|
| `POST /v1/compendium/monsters` | Creates monster → 201, body **without** tags; duplicate slug → 409; malformed → 400 |
| `GET /v1/compendium/monsters/<slug>` | Returns monster **with** tags; unknown → 404 |
| `POST /v1/compendium/items` | Creates item → 201; duplicate slug → 409; malformed → 400 |
| `GET /v1/compendium/items/<slug>` | Returns item; unknown → 404 |

All prior-stage endpoints (dice, checks, encounters, initiative, characters, combat, auth, storage) are untouched and verified working. `run.sh` is unchanged and ready to go. ✅
