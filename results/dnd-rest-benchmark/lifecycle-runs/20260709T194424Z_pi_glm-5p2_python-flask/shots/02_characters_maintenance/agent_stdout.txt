## Summary

I added the **Stage 1: Character Rules** endpoints to `app.py` while preserving all 8 core-stage routes untouched.

### New endpoints (all under `/v1/characters/`)

1. **`POST /ability-modifier`** — validates `score` is an int in `1..30` (rejecting bools/strings), returns `{"score", "modifier"}` where `modifier = (score - 10) // 2`. Python's `//` floors toward −∞, so score `9` → `−1` exactly as the spec requires.

2. **`POST /proficiency`** — validates `level` is an int in `1..20`, returns `{"level", "proficiency_bonus"}` computed as `2 + (level - 1) // 4`, which cleanly maps the four-level tiers (1‑4→2, 5‑8→3, … 17‑20→6).

3. **`POST /derived-stats`** — validates `level`, the six abilities (each `1..30`), and the `armor` object (`base`/`dex_cap` ints, `shield` bool). Reuses the same `ability_modifier` and `proficiency_bonus` helpers, then computes:
   - `hp_max = level * (6 + con_modifier)`
   - `armor_class = base + min(dex_modifier, dex_cap) + (2 if shield else 0)`

### Verification
