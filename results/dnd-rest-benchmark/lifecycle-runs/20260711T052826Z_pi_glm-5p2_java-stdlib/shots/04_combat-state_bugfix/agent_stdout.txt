## Root cause

The `combat-advance-expired-removed` test expected the `conditions` map to still contain the `"fighter"` key with an empty array `[]` after fighter's `blessed` condition ticked down to 0 and was removed:

```json
"conditions": { "fighter": [] }
```

But `conditionsMap` only emitted combatants whose condition list was **non-empty**:

```java
if (conds != null && !conds.isEmpty()) map.put(c.name, conditionList(conds));
```

So once fighter's last condition expired and the list became empty, the `"fighter"` key vanished entirely в†’ `"conditions":{}` в†’ evaluator reported `missing JSON key "fighter"`.

## The fix

Dropped the `!conds.isEmpty()` guard so every combatant is always emitted with their (possibly empty) condition list. This is safe because the evaluator's `jsonContains` does **subset matching** on objects вЂ” it only verifies that the `want` keys are present, never rejecting extra keys. The earlier advance tests only assert the `"fighter"` key, so the additional `rogue`/`mage` entries don't affect them.

```java
if (conds != null) map.put(c.name, conditionList(conds));
```
