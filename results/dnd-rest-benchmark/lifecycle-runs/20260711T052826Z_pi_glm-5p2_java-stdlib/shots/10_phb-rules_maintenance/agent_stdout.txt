All three new PHB endpoints match the spec exactly, and prior-stage endpoints still work. ✨

**Summary of changes to `Main.java`:**

1. **`WIZARD_SLOTS` static table** — hardcodes the PHB wizard spell-slot progression for level 5 (`{"1":4,"2":3,"3":2}`), looked up by level.

2. **Three new routes** in the `Router` switch:
   - `POST /v1/phb/spell-slots` → `phbSpellSlots`
   - `POST /v1/phb/rests/long` → `phbLongRest`
   - `POST /v1/phb/equipment-load` → `phbEquipmentLoad`

3. **Three handlers** following the existing codebase conventions (`parseObject` + strict `asInt` + `LinkedHashMap` for key ordering + `toJson`):
   - **Spell slots**: validates `class == "wizard"` and a supported `level`; returns the slot table.
   - **Long rest**: `hp_current → hp_max`; recovers `max(level/2 floor, 1)` hit dice (so `3 − 2 = 1`); reduces exhaustion by 1, floored at 0.
   - **Equipment load**: `capacity = strength * 15`; `encumbered = weight > capacity` (strictly exceeds, so weight == capacity is *not* encumbered).

Invalid requests (missing/non-integer fields, unsupported class/level) return `400`. `run.sh` is unchanged and still compiles + runs the server in the foreground on `127.0.0.1:$PORT`. 🐉
