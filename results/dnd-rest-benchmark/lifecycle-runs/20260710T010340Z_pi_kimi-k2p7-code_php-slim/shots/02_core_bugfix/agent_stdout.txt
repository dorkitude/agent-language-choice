Fixed, Chef! 🎲

**Problem:** The `/v1/dice/stats` endpoint was rounding the average to an integer:

```php
$average = (int) round($dice_count * ($sides + 1) / 2) + $modifier;
```

For `1d20-1`, the true average is `9.5`, but this rounded it to `10`.

**Fix in `index.php`:** Removed the rounding/cast so it returns the exact arithmetic mean:

```php
$average = $dice_count * ($sides + 1) / 2 + $modifier;
```

This preserves integer-looking results (`2d6+3` → `10`) while correctly emitting halves (`1d20-1` → `9.5`).

`./run.sh` is already in place and unchanged. The server is ready to start on `127.0.0.1:$PORT` in the foreground.
