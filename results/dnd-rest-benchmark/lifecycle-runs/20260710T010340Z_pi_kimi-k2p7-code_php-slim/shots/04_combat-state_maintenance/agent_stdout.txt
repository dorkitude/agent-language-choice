I added the three stateful combat endpoints to `index.php` while keeping every existing core + character endpoint intact:

- `POST /v1/combat/sessions` — creates a session with deterministic initiative sorting (`score` desc, `dex` desc, `name` asc) and returns `round`, `turn_index`, `active`, and `order`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition with positive `duration_rounds` to a valid combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances `turn_index`, wraps the round, and decrements/removes conditions only on the newly active combatant.

State lives in an in-memory `$sessions` array, which is fine for the lifetime of the server process. Invalid requests return `400`, and unknown session IDs return `404`.

`./run.sh` is unchanged and ready:

```bash
#!/usr/bin/env bash
set -euo pipefail
php -S 127.0.0.1:"$PORT" index.php
```
