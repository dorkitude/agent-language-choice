### What I added (`server.rb`)

Three new handlers, all reusing existing core math (`XP_TABLE`, `encounter_multiplier`, `THRESHOLDS`, `num`) and the SQLite compendium/campaign state:

1. **`POST /v1/dm/encounter-builder`** — looks up each monster's CR from the `compendium_monsters` table, computes base/adjusted XP via the same multiplier + threshold logic as `/v1/encounters/adjusted-xp`, classifies difficulty, and maps it to a deterministic `recommendation` (`easy` → `"safe warm-up"`). Returns 400 for malformed input, 404 for an unknown monster slug.

2. **`POST /v1/dm/loot-parcel`** — returns a deterministic tier-based parcel from a `TIER_LOOT` table (tier 1 → `coins_gp: 75`, `healing-potion ×2`). 400 for bad input.

3. **`POST /v1/dm/session-recap`** — reads the campaign's latest event summary from `campaign_events` (404 if the campaign doesn't exist) and derives `open_threads` deterministically from event summaries (`"Nyx scouts the goblin trail."` → `"Resolve goblin trail ambush"`).
