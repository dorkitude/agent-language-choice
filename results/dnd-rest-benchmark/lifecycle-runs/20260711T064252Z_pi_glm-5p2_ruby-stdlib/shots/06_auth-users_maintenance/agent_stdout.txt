## Changes to `server.rb`

**1. Stdlib password hashing (real hashing, no gems)**
- `require "openssl"` — uses `OpenSSL::KDF.pbkdf2_hmac` (PBKDF2-HMAC-SHA256, 100k iterations, random 16-byte salt).
- `hash_password` → stores a self-describing verifier `pbkdf2$<iter>$<salt_hex>$<hash_hex>`.
- `verify_password` → parses the verifier and uses `OpenSSL.fixed_length_secure_compare` (constant-time). Plain passwords are never stored or echoed.

**2. `POST /v1/auth/register`**
- Validates `username` against `\A[a-z0-9_-]{2,32}\z` → `400` if bad.
- Validates `password` is a string ≥ 8 chars → `400` if bad.
- Validates `role` ∈ `{dm, player}` → `400` if bad.
- Duplicate username → `409`.
- Success → `200 {"username": ..., "role": ...}`.

**3. `POST /v1/auth/login`**
- Malformed input (non-string/missing fields, unparseable JSON) → `400`.
- Bad credentials (unknown user or wrong password) → `401`.
- Success → `200 {"username": ..., "token": "session-<username>"}` (deterministic token).

**4. Routing & status map**
- Added `401`/`409` to `STATUS_TEXT`.
- Added the two new `when` cases in the router. All existing routes (health, dice, ability checks, encounters, initiative, characters, combat sessions/conditions/advance) are untouched and still pass.

## `run.sh` — unchanged
