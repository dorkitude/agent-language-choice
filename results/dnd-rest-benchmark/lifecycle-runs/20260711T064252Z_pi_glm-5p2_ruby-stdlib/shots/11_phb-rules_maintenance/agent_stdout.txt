## Summary

Added three deterministic PHB-rules endpoints to the Ruby stdlib server, preserving all prior-stage behavior:

**`POST /v1/phb/spell-slots`** — Returns wizard spell slots. Added a `WIZARD_SLOTS` table (levels 1–20); the benchmark-required wizard level 5 returns `{"1":4,"2":3,"3":2}`. Only non-zero slot levels are emitted. Validates `class == "wizard"` and integer level 1–20; anything else → 400.

**`POST /v1/phb/rests/long`** — Long-rest recovery:
- HP → `hp_max`
- Hit dice restored = `min(hit_dice_spent, max(level/2, 1))` (half level rounded down, minimum 1, never exceeding spent)
- Exhaustion reduced by 1, floored at 0

The spec example (level 5, 3 spent → restore 2 → 1 left; exhaustion 1 → 0) matches exactly.

**`POST /v1/phb/equipment-load`** — `capacity = strength * 15`, `encumbered = weight > capacity` (strictly exceeds, so equal weight is *not* encumbered).

All three return 400 with `{"error":"invalid request"}` on malformed/missing/non-JSON input, and all prior endpoints (health, dice, characters, combat, auth, compendium, campaigns, storage) remain intact and verified working. `run.sh` is executable and starts `ruby server.rb` in the foreground on `127.0.0.1:$PORT`. 🧙
