I added the auth-users stage to `main.go` while preserving every existing endpoint:

- `POST /v1/auth/register` — validates username format, password length, and role (`dm`/`player`), returns `{username, role}` on success, and `409` for duplicates.
- `POST /v1/auth/login` — returns `{username, token}` where `token` is `session-<username>` for good credentials, `401` otherwise.

Passwords are hashed through a small, isolated helper (`hashPassword`/`verifyPassword`) so a real production hash (bcrypt/argon2) can drop in later. No plaintext passwords are stored or echoed.

`run.sh` is unchanged and still starts the server in the foreground on `127.0.0.1:$PORT` using only the Go stdlib.
