I added the **Monster & Item Compendium** APIs to the existing Go stdlib D&D server while preserving every prior endpoint.

### What changed in `main.go`

- New `monsterEntry` and `itemEntry` types.
- New `Monsters` and `Items` maps in `dbState`, initialized on load/reset so storage reset stays cumulative.
- Store methods: `createMonster`, `getMonster`, `createItem`, `getItem`.
- New handlers:
  - `POST /v1/compendium/monsters` → creates monster, returns JSON without `tags`.
  - `GET /v1/compendium/monsters/{slug}` → returns full monster JSON with `tags`.
  - `POST /v1/compendium/items` → creates item.
  - `GET /v1/compendium/items/{slug}` → returns item.
- Duplicate slugs return `409`; missing records return `404`; malformed requests return `400`.
- Registered all four new routes alongside the existing ones.
