I added the **stateful combat** endpoints to the existing Go stdlib D&D API while keeping every prior endpoint intact:

- `POST /v1/combat/sessions` — creates a session with client-supplied `id`, computes initiative (`roll + dex`), sorts by score → dex → name, and starts at `round = 1`, `turn_index = 0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a timed condition to a named combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances the turn, wraps the round, and decrements conditions on the newly active combatant only.

All state is in-memory, guarded by a single mutex, and `./run.sh` still starts the server in the foreground on `127.0.0.1:$PORT`.

I smoke-tested the endpoints against the spec examples and they return the expected JSON, and `go build` / `go vet` are clean. No server is running now.
