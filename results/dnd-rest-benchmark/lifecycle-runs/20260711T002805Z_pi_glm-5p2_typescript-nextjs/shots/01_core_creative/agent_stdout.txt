## What I built

**Route handlers** (all under `app/`):
- `app/health/route.ts` — `GET /health` → `{"ok":true}`
- `app/v1/dice/stats/route.ts` — parses `<count>d<sides>[±<modifier>]`, validates positivity, returns `dice_count/sides/modifier/min/max/average`; 400 on invalid
- `app/v1/checks/ability/route.ts` — `total/modifier/dc` → `total/success/margin`
- `app/v1/encounters/adjusted-xp/route.ts` — XP-by-CR table, monster-count multipliers, per-level thresholds (level 3 required; full canonical DMG 1–20 included for robustness), summed party thresholds, difficulty classification
- `app/v1/initiative/order/route.ts` — sorts by score desc → dex desc → name asc

**Infrastructure fixes:**
- `app/layout.tsx` — added root layout (App Router needs it)
- `next.config.js` — converted `module.exports` → `export default {}` because `package.json` has `"type": "module"` (the seeded CommonJS config crashed on startup)
- `run.sh` — robust: ensures `node_modules/.bin/next` exists, defaults `PORT` to 3000, binds `127.0.0.1` via `exec` in foreground
