**What I added to `main.go`:**
- `POST /v1/auth/register` — validates username (`^[a-z0-9_-]{2,32}$`), password ≥8 chars, role `dm`/`player`; returns `409` on duplicate, `400` on malformed input, and `{username, role}` on success. 🔐
- `POST /v1/auth/login` — returns `{username, token: "session-<username>"}` on valid creds, `401` on bad creds.

**Password handling** 🧂 is isolated behind `hashPassword`/`verifyPassword` using salted SHA-256 with a constant-time compare. Go's stdlib has no adaptive KDF (bcrypt/argon2 live in `golang.org/x/crypto`, which the benchmark forbids), so I left a clear note to swap in a memory-hard hash for production. The plain password is never stored or echoed.

Verified live: register `200`, duplicate `409`, good login `200`, bad login `401`, bad username/short password/bad role all `400`, and `/health` still `200`. ✅
