I added the two auth endpoints to `src/server.ts` while preserving all existing D&D REST behavior:

- `POST /v1/auth/register` — validates username/password/role, hashes the password with `crypto.scryptSync` + random salt, rejects duplicates with `409`, and returns `{username, role}`.
- `POST /v1/auth/login` — validates input format, verifies the password against the stored hash, and returns `{username, token: "session-<username>"}` on success or `401` on bad credentials.

No plain passwords are stored or echoed. Password hashing is isolated in small helpers, so it could be swapped for a production option later if needed.

The project compiles cleanly:

```bash
npx tsc
```

`./run.sh` remains unchanged and will start the server in the foreground on `127.0.0.1:$PORT` when you run it. I did not start the server yet.
