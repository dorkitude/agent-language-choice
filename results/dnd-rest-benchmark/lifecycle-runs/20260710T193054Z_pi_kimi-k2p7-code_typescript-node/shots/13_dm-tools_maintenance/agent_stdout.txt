I added the three DM-facing endpoints to `src/server.ts` while preserving every existing route:

- **`POST /v1/dm/encounter-builder`** — looks up each monster slug in the compendium, reuses the existing adjusted-XP math, and returns `base_xp`, `adjusted_xp`, `difficulty`, `monster_count`, and a deterministic `recommendation`.
- **`POST /v1/dm/loot-parcel`** — validates the campaign exists and returns deterministic tier-1 loot (`75 gp` + 2 healing potions).
- **`POST /v1/dm/session-recap`** — reads the campaign’s events/characters and returns a deterministic summary plus `open_threads` (explicit `thread` events, or a generated thread from the latest narrative event).

I also refactored the adjusted-XP calculation into a shared `calculateEncounterXp` helper so both the core endpoint and the new DM tool use the exact same math.
