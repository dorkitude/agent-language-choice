**Design notes:**
- Deterministic, minimal code — CR/threshold/multiplier tables as frozen constants 🧊
- `average` and `adjusted_xp` render as integers when whole (matching `10` and `1700`), floats otherwise
- Invalid dice expressions / bad CRs / missing fields → HTTP 400
- Initiative sort: score desc → dex desc → name asc, exactly per spec
- Body parsing rewinds first so it's robust to how the client streams the request
