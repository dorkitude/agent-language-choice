All cases pass: registration returns `{username, role}`, duplicates → 409, malformed → 400, login returns `session-<username>`, bad/unknown creds → 401, and the prior-stage endpoints still work. ✨

`./run.sh` is unchanged and ready. Summary of what I added, Chef 🎲:

- **Routes** — `POST /v1/auth/register` and `POST /v1/auth/login`.
- **`Auth` module** — in-memory user store (mutex-guarded) with real hashing via `OpenSSL::PKCS5.pbkdf2_hmac` (SHA-256, per-user random salt, 100k iterations), isolated in `hash_password`/`password_matches?` so a production hash can swap in. Login uses constant-time comparison; the plaintext password is never stored or echoed.
- **Validation** — username `[a-z0-9_-]{2,32}`, password ≥ 8 chars, role ∈ {dm, player}; typed exceptions map cleanly to 400 / 409 / 401.

All core, character-rule, and combat-state endpoints are preserved untouched.
