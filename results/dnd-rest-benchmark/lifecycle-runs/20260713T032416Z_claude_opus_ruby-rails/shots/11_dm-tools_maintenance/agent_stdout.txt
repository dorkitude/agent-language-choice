## What I added

Three DM endpoints in `app.rb`, all preserving prior behavior:

**`POST /v1/dm/encounter-builder`** — groups `monster_slugs`, looks up each CR from the compendium (`goblin` → `1/4`), and **reuses `Dnd.adjusted_xp`** from the core suite. For the sample: base_xp 150, adjusted_xp 300 (×2 for 3 monsters), difficulty `easy` → recommendation `"safe warm-up"`. Unknown slug → 404. 🐉

**`POST /v1/dm/loot-parcel`** — deterministic tier table; tier 1 → 75 gp + 2 healing potions. Unknown tier → 400. 💰

**`POST /v1/dm/session-recap`** — reads durable campaign events; summary = latest event (`"Nyx scouts the goblin trail."`), and derives `open_threads` from scouting events (`"Resolve goblin trail ambush"`). Unknown campaign → 404. 📜
