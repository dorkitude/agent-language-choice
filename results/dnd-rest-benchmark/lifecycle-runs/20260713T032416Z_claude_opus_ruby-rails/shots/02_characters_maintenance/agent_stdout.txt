All endpoints match the spec exactly, invalid input returns 400, and the prior-stage `/v1/dice/stats` still works. ✨

`./run.sh` is unchanged and ready — it starts `rackup` in the foreground on `127.0.0.1:$PORT`. 🎲

What I added to `app.rb`:
- **`POST /v1/characters/ability-modifier`** — validates `score` (integer 1–30), `floor((score−10)/2)` via `fdiv(2).floor` so negatives floor correctly (9 → −1).
- **`POST /v1/characters/proficiency`** — validates `level` (1–20), bonus = `(level−1)/4 + 2`.
- **`POST /v1/characters/derived-stats`** — reuses the ability-modifier and proficiency logic, computes `hp_max`, `armor_class` (with dex cap + shield bonus), and per-ability modifiers.

All non-integer/out-of-range inputs raise `ArgumentError` and render `400`. 🛡️
