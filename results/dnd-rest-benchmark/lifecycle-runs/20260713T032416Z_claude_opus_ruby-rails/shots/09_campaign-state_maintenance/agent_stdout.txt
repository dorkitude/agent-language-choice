## Key subtlety I caught 🔍
The route `/v1/campaigns/:id/characters` puts the **campaign id** in `params[:id]`, which collides with the character's `id` in the JSON body. I read the body via `request.request_parameters` for character/event attrs so both ids survive — otherwise every character would have inherited the campaign's id.
