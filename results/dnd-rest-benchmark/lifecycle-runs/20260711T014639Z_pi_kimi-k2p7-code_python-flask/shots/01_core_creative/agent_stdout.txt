I implemented the core D&D REST engine in `app.py` with Flask 3.1.3 and Python 3.14.6:

- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` — parses `2d6+3` grammar, returns stats
- `POST /v1/checks/ability` — computes total, success, margin
- `POST /v1/encounters/adjusted-xp` — CR/XP tables, monster multipliers, party thresholds, difficulty rating
- `POST /v1/initiative/order` — sort by score, then dex, then name

I verified all endpoints via Flask's test client and they match the expected sample outputs. Installed `Flask==3.1.3` into `./.deps` to match the existing `PYTHONPATH` in `run.sh`.

`./run.sh` is ready and will start the server in the foreground on `127.0.0.1:$PORT`. Not starting the server now, per instructions.
