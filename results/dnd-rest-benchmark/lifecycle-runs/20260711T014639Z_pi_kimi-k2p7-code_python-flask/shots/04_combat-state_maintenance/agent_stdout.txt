I added the three combat-state endpoints to `app.py` while preserving all existing core and character-rule behavior:

- `POST /v1/combat/sessions` — creates an in-memory session with client-supplied ID, computes initiative (`roll + dex`), sorts by score desc → dex desc → name asc, and starts at `round=1`, `turn_index=0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition with a positive duration to a named combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances the turn, wraps the round, and decrements conditions only on the newly active combatant, removing them when they hit 0.

Validation:
- Duplicate/invalid session IDs → 400
- Unknown session IDs → 404
- Missing/invalid target, condition, or `duration_rounds` → 400

`./run.sh` is already in place and runs the Flask server in the foreground on `127.0.0.1:$PORT` using Python 3.14.6 and Flask 3.1.3. No new dependencies were needed.
