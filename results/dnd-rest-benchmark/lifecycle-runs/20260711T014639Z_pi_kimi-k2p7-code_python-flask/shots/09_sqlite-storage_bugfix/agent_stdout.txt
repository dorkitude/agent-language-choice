Fixed the bug in `app.py`! 🐛✨

**Problem:** `combat-advance-expired-removed` failed because the `conditions` response object only included combatants with active conditions. When the last condition on `fighter` expired, its key disappeared entirely.

**Fix:** In `/v1/combat/sessions/<id>/advance`, I now seed the `conditions` dict with every combatant in the initiative order as an empty array before populating active conditions:

```python
conditions = {row["name"]: [] for row in order}
for row in all_conditions:
    conditions.setdefault(row["target"], []).append(
        {"condition": row["condition"], "remaining_rounds": row["remaining_rounds"]}
    )
```

So now when a condition expires, `"fighter"` stays in the response as `"fighter": []`, matching the evaluator’s expectation. 🎯
