I added the three new combat-state endpoints to `app.rb` while preserving every existing route:

- `POST /v1/combat/sessions` — creates a session with initiative order (`roll + dex`, sorted by score desc, dex desc, name asc), `round=1`, `turn_index=0`.
- `POST /v1/combat/sessions/:id/conditions` — attaches a condition to a combatant with a positive `duration_rounds`.
- `POST /v1/combat/sessions/:id/advance` — advances the turn, wraps the index and increments the round, then decrements and removes expired conditions on the newly active combatant.

All state is in-memory (`COMBAT_SESSIONS`), malformed input returns 400, and unknown session IDs return 404. I kept `./run.sh` exactly as it was — it starts the server on `127.0.0.1:$PORT` in the foreground.

I also ran quick integration tests against the new endpoints and the existing core/character-rule endpoints; everything behaved correctly. No server is running now.
