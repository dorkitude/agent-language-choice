Fixed, Chef! 🎲

The bug was integer division in the dice stats `average` calculation. Ruby's `10 / 2` floors to an integer, so `1d20-1` was returning `9` instead of `9.5`.

Changed:

```ruby
average = (min + max) / 2
```

to:

```ruby
average = (min + max) / 2.0
```

`./run.sh` is already in place and ready to go:

```bash
#!/usr/bin/env bash
set -euo pipefail
bundle exec ruby app.rb -o 127.0.0.1 -p "$PORT"
```

No server started, and the existing behavior is preserved. 🚀
