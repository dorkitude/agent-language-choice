Done, Chef! 🍳 I added the auth-users stage to the existing Sinatra app while keeping all prior endpoints intact.

Changes in `app.rb`:
- Added `openssl` and `securerandom` to the requires.
- Added a small `PasswordHelper` using PBKDF2-HMAC-SHA256 (Ruby stdlib) so passwords are hashed and never echoed or stored in plain text.
- Added an in-memory `USERS` store with a mutex for thread safety.
- Added validation for `username`, `password`, and `role`.
- Added `POST /v1/auth/register` with `400`/`409` responses.
- Added `POST /v1/auth/login` returning deterministic `session-<username>` tokens and `401` for bad credentials.

`run.sh` is unchanged: it still starts the Puma server on `127.0.0.1:$PORT` in the foreground.
