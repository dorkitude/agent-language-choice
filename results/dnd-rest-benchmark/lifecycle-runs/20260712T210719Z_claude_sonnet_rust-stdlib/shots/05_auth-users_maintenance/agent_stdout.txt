Added `/v1/auth/register` and `/v1/auth/login` to `src/main.rs`, backed by a pure-std SHA-256 + salted iterated hash helper (`hash_password`/`verify_password`) so no plaintext passwords are stored or echoed. All prior endpoints (core, character-rules, combat-state) are untouched and verified via manual curl tests alongside the new auth flow (register, duplicate 409, invalid username/password 400, login success/401). `run.sh` already builds and runs the server in the foreground on `PORT`, so it's unchanged.
