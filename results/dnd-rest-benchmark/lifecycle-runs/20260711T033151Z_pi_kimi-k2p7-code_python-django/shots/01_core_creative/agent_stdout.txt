I implemented the core D&D REST engine in the seeded Django project:

- `dndsite/urls.py` now holds all five endpoints:
  - `GET /health` → `{"ok": true}`
  - `POST /v1/dice/stats` — parses `XdY±M` expressions, returns `dice_count`, `sides`, `modifier`, `min`, `max`, and `average`
  - `POST /v1/checks/ability` — computes `total`, `success`, `margin`
  - `POST /v1/encounters/adjusted-xp` — CR/XP table, count multipliers, party thresholds, and difficulty rating
  - `POST /v1/initiative/order` — sorts by score desc, dex desc, name asc

- `run.sh` kept its original shape but now uses `exec` so the server runs truly in the foreground.

I used `uv` to install the pinned Django 6.0.7 into `.deps` (Python dependency directory). Note: `uv` can’t directly manage the system Python 3.14.6 on this machine (`platform.mac_ver()` issue), so `run.sh` still launches the pinned `python3` directly with `PYTHONPATH=.deps`. That combination works and was verified against all sample requests:

- `GET /health` ✅
- `2d6+3` → dice stats match ✅
- ability check `9 + 5 vs 15` → `14 / false / -1` ✅
- adjusted XP sample → `850 / 3 / 2 / 1700 / deadly` ✅
- initiative tiebreaker → `rogue 17, ogre 15` ✅
