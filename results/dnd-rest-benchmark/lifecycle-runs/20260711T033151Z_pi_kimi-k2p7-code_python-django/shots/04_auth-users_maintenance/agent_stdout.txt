I added the auth stage to `dndsite/urls.py` without touching any existing endpoint behavior:

- **`POST /v1/auth/register`** — validates username (`2-32` lowercase letters/digits/`_`/`-`), password (`≥8` chars), and role (`dm`/`player`), hashes the password with Django's `make_password`, and returns `{"username": "...", "role": "..."}`. Duplicate usernames get `409`.
- **`POST /v1/auth/login`** — checks credentials with `check_password`, returns `{"username": "...", "token": "session-<username>"}`, and `401` for bad credentials.

Password hashing is isolated behind Django's standard `django.contrib.auth.hashers` helpers, and no plain password is ever stored or echoed.
