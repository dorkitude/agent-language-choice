I added the SQLite-backed campaign state APIs while preserving all existing endpoints. Here’s what changed:

**`dndsite/db.py`**
- Added `campaigns`, `characters`, and `events` tables to the schema.
- Enabled `PRAGMA foreign_keys = ON` on connections.
- Updated `storage_status` query and `reset_storage` drop order to account for the new tables.
- Added DB helpers: `create_campaign`, `get_campaign`, `create_character`, `list_characters`, `create_event`, `count_events`.

**`dndsite/urls.py`**
- Added views and routes:
  - `POST /v1/campaigns` → create campaign
  - `POST /v1/campaigns/<id>/characters` → add character
  - `POST /v1/campaigns/<id>/events` → add session log event
  - `GET /v1/campaigns/<id>/state` → read campaign state

**Behavior**
- Malformed input → `400`
- Unknown campaign → `404`
- Duplicate IDs → `409`
- `GET /v1/campaigns/<id>/state` returns the campaign, its characters list, and `log_count`.

`./run.sh` is already in place and unchanged—it initializes storage and runs the Django server on `127.0.0.1:$PORT` in the foreground.
