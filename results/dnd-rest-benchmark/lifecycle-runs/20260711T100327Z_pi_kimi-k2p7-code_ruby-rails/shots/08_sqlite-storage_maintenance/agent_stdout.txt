I added the SQLite-backed storage stage while preserving every existing D&D API endpoint.

What changed:
- **Gemfile** — added `sqlite3 ~> 2.0` and bundled it.
- **app.rb** — introduced a `GameDatabase` singleton that creates `game.db` on startup, initializes schema version `1`, and enables WAL mode.
- **Durable data moved to SQLite**:
  - `UserStore` now persists users in the `users` table.
  - `CombatSessionStore` now persists combat sessions in `combat_sessions`.
- **New endpoints**:
  - `GET /v1/storage/status` → `{"driver": "sqlite", "schema_version": 1, "initialized": true}`
  - `POST /v1/storage/reset` → drops benchmark tables, recreates schema, returns `{"ok": true, "schema_version": 1}`
- **config/routes.rb** — wired the new storage routes.

All prior endpoints (`/health`, dice, checks, encounters, initiative, characters, combat, auth) were tested and still pass. `./run.sh` is unchanged and will start the server on `127.0.0.1:$PORT` in the foreground. No server is running now.
