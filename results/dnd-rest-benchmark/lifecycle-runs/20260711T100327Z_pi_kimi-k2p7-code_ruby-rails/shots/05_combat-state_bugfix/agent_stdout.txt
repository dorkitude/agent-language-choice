**Root cause:** In `CombatController#advance`, the `conditions` hash only included combatants whose condition list was non-empty. When a condition expired, the combatant key disappeared entirely, which caused the failing test to complain about a missing `"fighter"` key.

**Fix:** Track whether each combatant has ever had a condition (via `has_conditions`), and include them in the `conditions` response if they have ever had conditions *or* currently have active ones. This preserves the prior behavior for untouched combatants while keeping the combatant key present with an empty array after all conditions expire.

Changed in `app.rb`:
- `CombatSessionStore.add_condition` now sets `combatant[:has_conditions] = true` when a condition is attached.
- `CombatController#advance` now builds the `conditions` hash using `if combatant[:has_conditions] || combatant[:conditions].any?`.
