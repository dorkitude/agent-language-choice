I added the **auth-users** stage to the existing Rails app without touching the prior endpoints:

- `POST /v1/auth/register` → validates username, password, and role, hashes the password with **BCrypt**, and returns `{"username": "...", "role": "..."}`.
- `POST /v1/auth/login` → checks the BCrypt hash and returns the deterministic `{"username": "...", "token": "session-<username>"}`.

Status codes match the spec: `400` for malformed input, `401` for bad credentials, `409` for duplicate usernames. Passwords are never echoed.

Files changed:
- `Gemfile` — added `bcrypt`
- `app.rb` — added `UserStore` and `AuthController`
- `config/routes.rb` — added the two auth routes
- `Gemfile.lock` / `.bundle/config` — updated for the vendored bcrypt install

`./run.sh` is ready and will start the server in the foreground on `127.0.0.1:$PORT` using Puma. I verified the existing endpoints and the new auth endpoints with Rack mock requests, but I didn’t start the actual HTTP server. 🚀
