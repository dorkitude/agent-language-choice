I created the core implementation in `./server.rb` using only Ruby’s standard library (`socket` + `json`), with no Rack, Sinatra, Rails, or external gems.

What’s implemented:
- `GET /health` → `{"ok":true}`
- `POST /v1/dice/stats` → parses `XdY+Z`, validates grammar, returns stats
- `POST /v1/checks/ability` → total, success, margin
- `POST /v1/encounters/adjusted-xp` → CR-to-XP table, count multiplier, level-3 thresholds, difficulty rating
- `POST /v1/initiative/order` → score, dex, name tie-breaking

I kept the existing `./run.sh` untouched:

```bash
#!/usr/bin/env bash
set -euo pipefail
ruby server.rb
```

The server reads `PORT` from the environment, binds to `127.0.0.1`, and runs in the foreground.

I tested all five endpoints and the error paths (invalid expression, unsupported level, 404) locally — they all matched the expected JSON responses. I’m not starting the server now, as requested.
