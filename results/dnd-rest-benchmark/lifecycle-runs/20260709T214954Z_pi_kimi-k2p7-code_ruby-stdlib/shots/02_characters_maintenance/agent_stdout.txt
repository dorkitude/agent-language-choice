I added the three character-rule endpoints to the existing Ruby stdlib server, leaving every core endpoint intact.

**Updated file:** `server.rb`

Added handlers:
- `POST /v1/characters/ability-modifier` — validates score `1..30` and returns `floor((score - 10) / 2)`, including negative halves correctly.
- `POST /v1/characters/proficiency` — validates level `1..20` and returns the tiered proficiency bonus.
- `POST /v1/characters/derived-stats` — computes all six modifiers, proficiency bonus, `hp_max`, and `armor_class` with the shield/dex-cap rules.

All success responses are JSON, and invalid inputs return `400` with `{"error":"bad request"}`.

**Run script:** `run.sh` is unchanged and ready:
```bash
#!/usr/bin/env bash
set -euo pipefail
ruby server.rb
```

It starts the server in the foreground on `127.0.0.1` using the `PORT` env var.

I spot-tested the new endpoints plus the old `/health` and `/v1/dice/stats` endpoints locally, and the responses matched the spec. No server is running now.
