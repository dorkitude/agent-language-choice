## Summary

Added three new endpoints to `Main.java` (Java stdlib `com.sun.net.httpserver`):

| Endpoint | Behavior |
|---|---|
| `POST /v1/characters/ability-modifier` | Validates `score` ∈ [1,30], returns `modifier = floorDiv(score-10, 2)` (floors negative halves correctly: 9 → -1) |
| `POST /v1/characters/proficiency` | Validates `level` ∈ [1,20], returns tiered bonus (2/3/4/5/6) |
| `POST /v1/characters/derived-stats` | Computes proficiency, `hp_max = level*(6+con_mod)`, `armor_class = base + min(dex_mod, dex_cap) + shield_bonus`, and all six ability modifiers |

Key implementation details:
- Used `Math.floorDiv` for correct floor division on negative halves.
- Added a strict `requireInt` helper that rejects missing/non-integer values with `400` (only accepts true JSON integers, plus whole-number doubles for robustness).
- `asBool` defaults missing `shield` to `false` (shield_bonus 0).
- All field ordering preserved via `LinkedHashMap` so JSON output matches the spec exactly.

`./run.sh` is already in place and correct (compiles + runs `java Main` in foreground, listens on `127.0.0.1:$PORT`). Regression-tested all 8 core endpoints — still green. 🟢
