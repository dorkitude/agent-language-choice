**Root cause:** `run.sh` was calling `next dev` directly, but `next` wasn't on `PATH` (it only lives in `node_modules/.bin`). So the server never started.

**What I changed:**
- `run.sh` now uses `exec npx next dev -H 127.0.0.1 -p "$PORT"` so it resolves correctly.
- Implemented the missing route handlers:
  - `POST /v1/dice/stats`
  - `POST /v1/checks/ability`
  - `POST /v1/encounters/adjusted-xp`
  - `POST /v1/initiative/order`
- Updated `tsconfig.json` to be compatible with Next.js 16.2.10 (`moduleResolution: bundler`, `jsx: react-jsx`, etc.) so the type-checker passes.

I also did a quick smoke test: all endpoints return the expected responses from the spec, and the server runs in the foreground on `127.0.0.1:$PORT` as required.
