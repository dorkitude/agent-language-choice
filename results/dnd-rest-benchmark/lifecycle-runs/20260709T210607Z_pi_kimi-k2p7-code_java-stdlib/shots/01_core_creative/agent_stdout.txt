I created `Main.java` implementing a minimal D&D REST engine using only the Java standard library (`com.sun.net.httpserver.HttpServer`). It handles all five required endpoints:

- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` → parses `XdY+/-Z` expressions and returns stats
- `POST /v1/checks/ability` → computes total, success, margin
- `POST /v1/encounters/adjusted-xp` → computes base XP, monster multiplier, adjusted XP, and difficulty
- `POST /v1/initiative/order` → sorts combatants by score, dex, name

I also included a tiny custom JSON parser/serializer so no external dependencies are needed.

The existing `run.sh` is preserved: it compiles `Main.java` and runs `java Main` in the foreground, listening on `127.0.0.1:$PORT`.

I verified it with OpenJDK 26.0.1 and curl-tested all endpoints against the spec samples — they match exactly. 🍳
