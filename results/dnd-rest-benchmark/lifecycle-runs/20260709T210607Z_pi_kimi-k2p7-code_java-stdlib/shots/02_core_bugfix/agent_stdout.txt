**What was wrong:** The `/v1/dice/stats` endpoint calculated `average` using integer division:

```java
int average = (min + max) / 2;
```

For `1d20-1`, that produced `9` instead of `9.5`.

**The fix:** Changed it to use floating-point division and format whole numbers cleanly:

```java
double average = (min + max) / 2.0;
...
.append(",\"average\":").append(formatNumber(average))
```
