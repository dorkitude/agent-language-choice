Found the culprit in `handleDerivedStats` — it was appending an extra top-level `}` after the modifiers object, producing invalid JSON like `...}}}`.

I changed this in `Main.java`:

```java
// before
.append("}}")
.append("}");

// after
.append("}}");
```

Then verified it compiles and the derived-stats response parses cleanly:

```json
{"level":5,"proficiency_bonus":3,"hp_max":35,"armor_class":16,"modifiers":{"str":3,"dex":2,"con":1,"int":-1,"wis":1,"cha":0}}
```

`./run.sh` is still in place and ready to start the server on `PORT`. No other behavior was changed. 🍳
