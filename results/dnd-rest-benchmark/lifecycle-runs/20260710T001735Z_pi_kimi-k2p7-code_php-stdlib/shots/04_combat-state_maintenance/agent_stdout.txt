I added the **combat-state** endpoints to `index.php` and kept every existing `core` + `characters` endpoint untouched:

- `POST /v1/combat/sessions` — creates a client-ID’d session, sorts initiative, and returns the first active combatant.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition to a named combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances the turn index, wraps the round, and decrements conditions on the newly active combatant.

State is stored in a tiny JSON file (`combat_state.json`) with file locking, so it survives PHP’s built-in server request cycle. `run.sh` now wipes that file on startup so each server process starts fresh.
