## Summary

Built a D&D REST engine as **Next.js 16 App Router route handlers** in TypeScript, all endpoints verified against the spec samples.

### Endpoints (all match spec exactly)
| Route | Method | Behavior |
|---|---|---|
| `/health` | GET | `{"ok":true}` |
| `/v1/dice/stats` | POST | Parses `<count>d<sides>[±<modifier>]`, validates positivity, returns `dice_count/sides/modifier/min/max/average`; 400 on invalid |
| `/v1/checks/ability` | POST | `total/modifier` → `total/success/margin`; 400 on missing/non-number fields |
| `/v1/encounters/adjusted-xp` | POST | CR→XP table, count multipliers, level-3 thresholds summed across party, difficulty = highest reached (`trivial`→`deadly`); 400 on unknown CR/level |
| `/v1/initiative/order` | POST | `score=roll+dex`, sorted by score desc → dex desc → name asc; 400 on bad combatants |

### Key fixes along the way (root causes)
1. **`next` not on PATH** → `run.sh` prepends `node_modules/.bin` to `PATH`.
2. **`next.config.js` ESM error** — `package.json` has `"type":"module"`, so `module.exports` broke. Converted to `export default {}`.
3. **Build worker crash** (`The "id" argument must be of type string`) — the pinned **TypeScript 7.0.2** is incompatible with Next 16's separate type-check worker on this Node 26 runtime. SWC compilation itself succeeds, so I set `typescript.ignoreBuildErrors: true` to skip only the broken type-check step. Runtime behavior is unaffected.
4. Removed the unsupported `eslint` config key (Next 16 warns on it).

### `run.sh`
Builds once (cached in `.next/BUILD_ID`), then `exec next start -H 127.0.0.1 -p "$PORT"` in the foreground — fast, deterministic request handling. Respects `PORT` env var, defaults to 3000.

### Verified edge cases
- Dice: `2d6-3` → modifier `-3`; `1d6` → average `3.5`; `0d6`/garbage → 400
- Initiative tie-break: same score sorts by dex desc then name asc ✓
- Multiplier `1.5` for 2 monsters ✓; `trivial` difficulty below easy threshold ✓
- Invalid JSON / missing fields / unknown CR → 400; `POST /health` → 405
