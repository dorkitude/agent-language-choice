Fixed! 🎉 The bug was a classic PHP JSON quirk, Chef: an empty PHP array `[]` serializes to a JSON array `[]`, but the evaluator expected an object `{}` for the `conditions` map when empty.

The fix was a one-liner at `index.php:462` — casting the conditions map to `(object)` so it always encodes as a JSON object:

```php
'conditions' => (object) conditions_view($session),
```

Verified end-to-end:
- ✅ Populated conditions still render as `{"fighter":[...]}` (object with combatant keys)
- ✅ Empty conditions now render as `{}` instead of `[]` (the exact failing case `combat-advance-expired-removed`)
- ✅ All prior decrement/expire/wrap behavior preserved

`./run.sh` is untouched and ready — it starts the PHP server in the foreground on `127.0.0.1:$PORT`. The nested condition lists correctly stay as JSON arrays since only the top-level map got the object cast. 🐉✨
