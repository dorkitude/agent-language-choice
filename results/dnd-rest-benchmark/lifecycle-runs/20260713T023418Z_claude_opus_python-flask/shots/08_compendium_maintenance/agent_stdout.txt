Added SQLite-backed compendium APIs to the existing Flask app while preserving every prior endpoint:

- **`POST /v1/compendium/monsters`** — validates slug/name/cr/armor_class/hit_points/tags, `409` on duplicate slug, response omits `tags` (per spec)
- **`GET /v1/compendium/monsters/<slug>`** — returns full record including `tags`, `404` if unknown
- **`POST /v1/compendium/items`** — validates fields, `409` on duplicate slug
- **`GET /v1/compendium/items/<slug>`** — `404` if unknown

Two new tables (`monsters`, `items`) are created in `init_db()` (IF NOT EXISTS, so the existing `game.db` upgrades cleanly) and dropped in `reset_db()` alongside the others. Malformed input → `400`, unknown → `404`, dup slug → `409`. Tags are stored as JSON.

Verified all four new endpoints end-to-end plus the prior `/health` route, then wiped my test rows from `game.db`. `./run.sh` is unchanged and ready. 🐍
