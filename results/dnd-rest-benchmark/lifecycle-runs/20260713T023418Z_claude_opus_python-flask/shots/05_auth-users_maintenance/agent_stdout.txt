**What I added** (to `app.py`, preserving every existing endpoint):

- `POST /v1/auth/register` — validates username (`^[a-z0-9_-]{2,32}$`), password (≥8 chars), role (`dm`/`player`); returns `409` on duplicates; stores a **werkzeug** password hash (Flask's bundled, framework-provided option — no plaintext stored or echoed).
- `POST /v1/auth/login` — verifies against the hash, returns `{"username", "token": "session-<username>"}`, or `401` on bad/unknown credentials.

Malformed input → `400`, bad credentials → `401`, duplicate username → `409`. `run.sh` was already present and correct (starts Flask in the foreground on `127.0.0.1:$PORT`), so no changes needed there. I did **not** start the server. 🐉
