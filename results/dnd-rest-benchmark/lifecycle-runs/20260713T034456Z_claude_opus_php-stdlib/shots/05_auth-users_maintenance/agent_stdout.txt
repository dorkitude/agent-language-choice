## What I added 🔐

Two new endpoints appended to `index.php`, following the existing per-PID temp-file store pattern used by combat sessions:

**`POST /v1/auth/register`**
- Validates `username` (`^[a-z0-9_-]{2,32}$`), `password` (≥8 chars), `role` (`dm`/`player`) → `400` on malformed input
- Duplicate username → `409`
- Returns `{"username", "role"}` — never echoes the password

**`POST /v1/auth/login`**
- Bad/unknown credentials → `401`
- Success → `{"username", "token": "session-<username>"}`

## Security note 🛡️
Passwords are hashed with PHP's built-in `password_hash(..., PASSWORD_DEFAULT)` and checked with `password_verify()` — a real, strong standard hash. I isolated it behind `hash_password()` / `verify_password()` helpers so it's swappable, and the stored hash never appears in any response.

All prior-stage behavior (core, characters, combat-state) is untouched, and `run.sh` already starts the built-in server on `127.0.0.1:$PORT` in the foreground — no changes needed there. I did not start the server, per the contract. 🎲
