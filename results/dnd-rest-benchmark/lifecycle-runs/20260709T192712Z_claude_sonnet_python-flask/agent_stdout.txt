The fix: in `advance_combat_turn`, the `conditions_by_name` dict was only including combatants whose condition list was non-empty (`if c["conditions"]`), so when the active combatant's last condition expired this turn, it dropped out of the response entirely instead of showing `[]`. Changed the filter to `if c["conditions"] or c is active` so the currently-active combatant's key always appears (even empty), while other combatants still only appear when they have live conditions — matching both the passing tests and the previously-failing one.

`./run.sh` is unchanged and ready; no other endpoints were touched.
