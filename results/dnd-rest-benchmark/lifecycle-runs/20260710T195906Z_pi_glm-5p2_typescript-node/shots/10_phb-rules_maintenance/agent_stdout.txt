## Summary

I inherited the existing D&D REST API (TypeScript 7.0.2 / Node 26 `http` stdlib, no frameworks) and added the **Stage 7 — Selected PHB Rules** endpoints as pure stateless handlers in `src/server.ts` (no storage changes needed):

| Endpoint | Behavior |
|---|---|
| `POST /v1/phb/spell-slots` | Wizard level 5 → `{"class":"wizard","level":5,"slots":{"1":4,"2":3,"3":2}}`; other class/level → 400 |
| `POST /v1/phb/rests/long` | hp→hp_max; hit dice recovered = `max(1, floor(level/2))`, spent clamps at 0; exhaustion `max(0, −1)` |
| `POST /v1/phb/equipment-load` | capacity = `strength*15`; `encumbered` = `weight > capacity` (strictly greater) |

**Validation** — non-object body, wrong types, out-of-range ints, and unsupported class/level all return `400` (non-2xx). Successful responses are all JSON.

**Preserved behavior** — all prior routes (`/health`, `/v1/storage/*`, dice, checks, encounters, initiative, characters, combat sessions, auth, compendium, campaigns) are untouched; regressions confirmed passing.

**Build/run** — `tsc` compiles clean with the pinned `typescript@7.0.2`; `./run.sh` (already executable) runs `tsc` then `node dist/server.js` in the foreground, listening on `127.0.0.1:$PORT`. I did not start the server in my final answer.

`./run.sh` is ready. ✅
