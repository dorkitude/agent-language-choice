## Root cause

The `auth-register-user` test expected **HTTP 201 Created** for a successful registration, but `handleRegister` in `src/server.ts` returned `200 OK`. The response body (`{"username":"dm","role":"dm"}`) was already correct — only the status code was off.

## Fix

One-line change in `src/server.ts`:

```ts
USERS.set(username, record);
return { status: 201, body: { username, role } };  // was 200
```
