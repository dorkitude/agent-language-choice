## Summary

I added the **DM Tools** stage (Maintenance Stage 8) to the Go stdlib D&D REST API, preserving all 37 prior-stage endpoints.

### What I added

Three new routes registered in `main()` and implemented at the end of `main.go`:

1. **`POST /v1/dm/encounter-builder`** — Looks up each monster's CR from the stored compendium, then reuses the core-suite adjusted-XP math (`crXP` table, `multiplierFor`, and the level-3 threshold constants) to compute `base_xp`, `adjusted_xp`, `difficulty`, and a deterministic `recommendation` (`easy` → `"safe warm-up"`).

2. **`POST /v1/dm/loot-parcel`** — Returns deterministic per-tier loot from a fixed `tierLoot` table (tier 1 → `coins_gp: 75`, `items: [{healing-potion, 2}]`). The `seed` is accepted but output is fixed per the spec.

3. **`POST /v1/dm/session-recap`** — Derives `summary` from the campaign's most recent session-log event and returns a deterministic `open_threads` list. Unknown campaign → 404.

### Validation
- Invalid requests return non-2xx: unknown monster slug → 404, unknown campaign → 404, bad tier → 400, empty `campaign_id` → 400.

`./run.sh` is unchanged and starts the server in the foreground on `127.0.0.1:$PORT`. No third-party packages added — pure `net/http` + `encoding/json`. 🛡️
