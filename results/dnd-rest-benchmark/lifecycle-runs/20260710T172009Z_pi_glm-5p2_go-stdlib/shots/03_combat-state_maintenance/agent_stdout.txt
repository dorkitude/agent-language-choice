## What I added to `main.go`

Three new routes (registered alongside the existing 8 endpoints, none of which were touched):

1. **`POST /v1/combat/sessions`** — stores an in-memory session. Computes `score = roll + dex`, sorts by score desc → dex desc → name asc (reusing the same tie-break logic as `/v1/initiative/order`), starts `round=1`, `turn_index=0`, returns `active` + `order`.
2. **`POST /v1/combat/sessions/{id}/conditions`** — attaches a condition to a named combatant. `400` for unknown target / non-positive `duration_rounds`, `404` for unknown session.
3. **`POST /v1/combat/sessions/{id}/advance`** — bumps `turn_index`, wraps + increments `round` at the end, ticks down the *active* combatant's conditions at the start of their turn, removes any hitting `0`, and returns the full `conditions` map (only combatants that still have conditions).
