## What I added

Inherited the existing Go stdlib D&D REST API (core suite 8/8 passing) and added **three new character endpoints** to `main.go`, preserving all prior behavior:

| Endpoint | Behavior |
|---|---|
| `POST /v1/characters/ability-modifier` | Validates `score` ∈ [1,30] → `modifier = floor((score-10)/2)` with correct negative-half flooring (score 9 → -1) |
| `POST /v1/characters/proficiency` | Validates `level` ∈ [1,20] → tiered bonus 2→6 |
| `POST /v1/characters/derived-stats` | Validates level + all 6 ability scores; computes modifiers, proficiency, `hp_max = level*(6+con_mod)`, `armor_class = base + min(dex_mod, dex_cap) + shield_bonus` |

## Key implementation notes

- **Floor division**: Go's `/` truncates toward zero, so I added an explicit adjustment (`if diff < 0 && diff%2 != 0 { m-- }`) to correctly floor negative halves — score 8 and 9 both yield -1. ✅
- **Validation**: Out-of-range `score`/`level` and invalid JSON return `400`, matching the spec. Ability scores in derived-stats are validated 1-30 for consistency with the standalone endpoint.
- **No third-party deps** — pure `net/http` + `encoding/json`, Go 1.26.5.
