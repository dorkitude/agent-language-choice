## Summary

**Root cause:** In `handleAdvanceTurn`, when a combatant's last condition expired (remaining rounds reached 0), the code did `delete(sess.Conditions, activeName)` вЂ” wiping the combatant's key from the conditions map entirely. This produced `"conditions":{}` instead of the expected `"conditions":{"fighter":[]}`.

The spec says to "Remove a **condition** when its remaining duration reaches 0" вЂ” the condition entry, not the combatant's slot. The evaluator's `combat-advance-expired-removed` test expects the fighter's key to remain visible (with an empty array) as evidence the condition was removed.

**Fix** (in `main.go`, `handleAdvanceTurn`):
- Replaced `delete(sess.Conditions, activeName)` with `sess.Conditions[activeName] = kept` вЂ” keeping the key with a non-nil empty slice that serializes to `[]`.
- Added a `hadConds` guard (via the two-value map lookup `conds, hadConds := sess.Conditions[activeName]`) so we only retain the key for combatants who **actually had conditions**. Combatants who never had conditions (like mage/rogue on their turns) still don't appear in the response вЂ” preserving the passing tests.
