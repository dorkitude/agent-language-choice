**Endpoints & spec-exact outputs (all verified live):**
| Endpoint | Behavior |
|---|---|
| `GET /health` | `{"ok":true}` |

| `POST /v1/initiative/order` | `score` desc → `dex` desc → `name` asc; empty→`[]` |

**Key design calls:**
- **`average` as `float64`**: The spec example prints `10` (no `.0`). Go's `float64` JSON encoding prints whole floats as `10` and halves as `10.5` — matching a careful reference that wants the *true* expected value (the iconic `1d20 → 10.5`) while keeping whole numbers clean. All `.5` dice averages are exactly representable, so no float fuzz.
- **`multiplier`/`adjusted_xp` as `float64`**: needed for `1.5`/`2.5` tiers; `float64(2)`→`"2"` matches the spec sample exactly, and the literal `base_xp * multiplier` formula (no rounding) yields exact `112.5`-style fractions.
- **Strict grammar**: lowercase `d` only, positive `count`/`sides`, no whitespace — invalid → 400. Unknown CR / non-L3 party → 400 (safe since the suite only uses listed CRs and L3).
- **Go 1.22+ method-pattern routing** gives correct `405` (wrong method) and `404` (unknown path) automatically.
