**What I added** to `server.py` (stdlib only — `http.server` + `sqlite3` + `json`):

- `POST /v1/campaigns` → create campaign (409 on duplicate id)
- `POST /v1/campaigns/{id}/characters` → add character (404 unknown campaign, 409 dup char id)
- `POST /v1/campaigns/{id}/events` → append session log event, returns `{id, kind}`
- `GET /v1/campaigns/{id}/state` → full state with `characters` roster + `log_count`

**Design notes:**
- Followed the existing write-through pattern: in-memory `CAMPAIGNS` dict as working set, persisted to the SQLite `kv` table under a `"campaigns"` namespace. State survives restarts (verified). 💾
- Wired into `storage_load`/`storage_reset` alongside the other namespaces.
- Validation returns `400` malformed, `404` unknown, `409` duplicates — matching the contract.
