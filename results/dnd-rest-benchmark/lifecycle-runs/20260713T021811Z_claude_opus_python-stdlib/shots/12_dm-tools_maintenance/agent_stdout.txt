## What I added 🎲

Three new DM-facing endpoints in `server.py`, all reusing existing state — no new persistence:

- **`POST /v1/dm/encounter-builder`** — looks up each monster's CR from the compendium store, then delegates straight to the existing core `adjusted_xp()` math. Returns `base_xp`/`adjusted_xp`/`difficulty`/`monster_count` plus a deterministic `recommendation` keyed off the difficulty band (`easy → "safe warm-up"`). Unknown monster slug → `404`.
- **`POST /v1/dm/loot-parcel`** — deterministic tier table; `seed` is accepted but ignored so output is reproducible. Invalid/unknown tier → `400`.
- **`POST /v1/dm/session-recap`** — echoes `campaign_id`, derives `summary` from the campaign's most recent event when present (falls back to the documented default), and returns the open thread.

Routing: added a `DM_ROUTES` dict wired into `do_POST` (returns `200` on success, propagates `ApiError` status on invalid input). The 404-guard was updated so unknown paths still 404.

## Verified ✅
- All three endpoints return **exactly** the spec's example JSON.
- Invalid tier → `400`, unknown monster → `404` (non-2xx as required).
- `/health` and prior endpoints untouched — cumulative suite preserved.
