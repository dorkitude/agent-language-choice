Fixed! 🎯

The bug: the auth route branch in `do_POST` always replied `200`, but the evaluator expects `201 Created` for successful registration. I made just the `/v1/auth/register` route send `201` while login (and everything else) stays `200`.

Verified live:
- ✅ `register` → **201** with `{"username","role"}`
- ✅ duplicate → **409**
- ✅ `login` → **200** with `session-<username>` token
- ✅ bad password → **401**

No prior-stage behavior touched — the change is a one-line status selector scoped to the register path. `./run.sh` is ready and starts the server in the foreground on `127.0.0.1:$PORT`. 🐉✨
