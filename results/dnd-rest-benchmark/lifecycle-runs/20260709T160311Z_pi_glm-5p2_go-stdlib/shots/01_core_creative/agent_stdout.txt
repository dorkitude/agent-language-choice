### Key design decisions (rooted in the spec's number formatting)

- **`average` as int**: spec shows bare `10` (not `10.0`), and `(min+max)/2` integer division is mathematically identical to `count*(sides+1)/2+mod` for integers в†’ matches `2d6+3 в†’ 10` exactly.
- **`multiplier`/`adjusted_xp` as `float64`**: Go's json renders whole floats without decimals (`2.0в†’"2"`, `1700.0в†’"1700"`) вЂ” which matches the spec's bare `2` and `1700`, while still supporting `1.5`/`2.5` and fractional results like `52.5`.
- **Strict dice regex** `^([1-9][0-9]*)d([1-9][0-9]*)(?:([+-])([0-9]+))?$` вЂ” rejects `0d6`, `2d6+`, leading zeros, trailing junk, all в†’ 400.
