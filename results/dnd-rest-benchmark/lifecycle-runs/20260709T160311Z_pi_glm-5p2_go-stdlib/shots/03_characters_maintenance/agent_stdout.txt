All done! Here's a summary of what I added to the existing D&D REST API 🐉

## New endpoints (stage: characters)

1. **`POST /v1/characters/ability-modifier`** — validates score (1–30), returns `{"score", "modifier"}` using `floor((score-10)/2)`. Uses a custom `floorDiv` helper because Go's integer division truncates toward zero (so score 9 → -1, not 0).

2. **`POST /v1/characters/proficiency`** — validates level (1–20), returns `{"level", "proficiency_bonus"}` with the standard tiered bonus (2/3/4/5/6).

3. **`POST /v1/characters/derived-stats`** — validates level + all six ability scores, computes proficiency, all six modifiers, `hp_max = level * (6 + con_mod)`, and `armor_class = base + min(dex_mod, dex_cap) + shield_bonus`.

## Key details
- **Floor correctness**: The `floorDiv` helper corrects Go's truncation-toward-zero division for negative dividends — critical for the ability modifier formula (score 9 → -1, score 7 → -2).
- **Validation**: Out-of-range scores/levels and non-integer JSON values all return `400`.
- **Preserved behavior**: All 8 core-suite endpoints untouched and verified still passing (including the 1d20-1 average=9.5 fix from the previous bugfix shot).
- **No third-party deps** — pure `net/http` + `encoding/json`, Go 1.26.5 stdlib only.
- `./run.sh` starts the server in the foreground on `127.0.0.1:$PORT` ✅
