I implemented the core D&D REST engine in `app.py` using **Flask 3.1.3** and **Python 3.14.6**, and kept `./run.sh` ready to start the server in the foreground on `127.0.0.1` via the `PORT` env var.

Implemented endpoints:
- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` → parses `XdY+Z` / `XdY-Z` expressions, returns dice count, sides, modifier, min, max, and average
- `POST /v1/checks/ability` → computes total, success, margin
- `POST /v1/encounters/adjusted-xp` → uses the CR/XP table, monster count multiplier, and level-3 party thresholds
- `POST /v1/initiative/order` → sorts by score, then dex, then name

I also installed the dependencies into `.deps` so `run.sh`’s `PYTHONPATH` can find them, and verified the sample requests return the expected responses. I’ve stopped the test server and am not starting it now.
