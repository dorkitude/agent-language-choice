I preserved all existing endpoints and added the three new character-rule endpoints to `app.py`:

- `POST /v1/characters/ability-modifier`
- `POST /v1/characters/proficiency`
- `POST /v1/characters/derived-stats`

Key implementation details:
- Ability modifier uses integer division `(score - 10) // 2`, which correctly floors negatives (e.g., score 9 → −1).
- Proficiency bonus maps level ranges 1–4/5–8/9–12/13–16/17–20 to 2/3/4/5/6.
- Derived stats validates the full ability set, computes modifiers, proficiency, `hp_max`, and `armor_class` with shield bonus.
- Invalid inputs return `400` with a JSON error.

I verified the new endpoints with Flask’s test client, and `./run.sh` remains in place and executable. I did not start the server. 🍳
