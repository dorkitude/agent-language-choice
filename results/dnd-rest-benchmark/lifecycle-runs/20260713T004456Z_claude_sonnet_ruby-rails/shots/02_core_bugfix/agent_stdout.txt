Root cause was that `DndApp.initialize!` was called *after* `routes.draw`, which triggers Rails' routes reloader — since there's no `config/routes.rb` file on disk, the reloader wiped the manually-drawn routes down to just the default `/rails/info` routes, leaving `/health` and friends 404ing. 🐛

Fixed by calling `initialize!` **before** drawing routes in app.rb:12. Verified all five endpoints (`/health`, dice stats, ability check, adjusted XP, initiative order) plus the 400 invalid-expression case — all match spec exactly. `./run.sh` is unchanged and ready. ✨
