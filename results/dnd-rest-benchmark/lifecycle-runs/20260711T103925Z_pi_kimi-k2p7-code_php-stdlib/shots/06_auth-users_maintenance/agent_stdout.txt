Added the auth stage to the PHP stdlib D&D API:

- **`POST /v1/auth/register`** — validates username/password/role, returns 409 on duplicates, and stores a **bcrypt** password hash via PHP’s `password_hash()`.
- **`POST /v1/auth/login`** — verifies credentials with `password_verify()`, returns 401 on failure, and emits the deterministic token `session-<username>`.

All existing endpoints (`/health`, dice, checks, encounters, initiative, character rules, and combat state) remain untouched. 

Updated `run.sh` to also reset the new `.users.json` file so each run starts clean. Server is **not** started — `./run.sh` is ready when you are.
