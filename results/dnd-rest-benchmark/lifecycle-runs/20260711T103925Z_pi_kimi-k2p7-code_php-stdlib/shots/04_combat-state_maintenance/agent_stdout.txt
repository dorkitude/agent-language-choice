I added the new combat-state endpoints to `index.php` and updated `run.sh` to clear the local state file before starting:

- `POST /v1/combat/sessions` — creates a session with initiative order (score, then dex, then name) and round/turn tracking.
- `POST /v1/combat/sessions/{id}/conditions` — attaches conditions to a combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances the turn, wraps the round, and ticks down conditions on the newly active combatant.

State is persisted in a local `.combat-state.json` file (since PHP’s built-in server resets state between requests), and `run.sh` wipes it on startup so each server run starts fresh.

`./run.sh` is ready and will start the server on `127.0.0.1:$PORT` in the foreground. No server is running now.
