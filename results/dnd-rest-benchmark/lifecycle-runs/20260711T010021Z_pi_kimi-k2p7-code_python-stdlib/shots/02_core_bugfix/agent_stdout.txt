The bug was in `server.py`’s dice average calculation: it used `//` (integer floor division), so `1d20-1` returned `9` instead of `9.5`.

I changed it to true division `/` and convert back to `int` only when the result is a whole number, keeping the existing integer response for cases like `2d6+3`.

- `run.sh` is unchanged and ready.
- `server.py` passes `python3 -m py_compile`.

No server was started during the fix.
