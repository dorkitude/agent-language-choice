I added the three combat-state endpoints to `server.py` while preserving every existing core and character-rule endpoint:

- `POST /v1/combat/sessions` — creates a session with client-supplied `id`, computes initiative (`roll + dex`), sorts by score → dex → name, and sets `round = 1`, `turn_index = 0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition to a named combatant with a positive `duration_rounds`.
- `POST /v1/combat/sessions/{id}/advance` — advances `turn_index`, wraps to `0` while incrementing `round`, and decrements/removes conditions only on the newly active combatant.

State lives in the in-memory `SESSIONS` dict. Missing sessions return `404`; malformed input returns `400`. `run.sh` remains unchanged and runs the server in the foreground on `127.0.0.1:$PORT` using Python stdlib only.
