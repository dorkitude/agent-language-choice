**What I added to `server.py`:**

- **Password hashing helpers** using only the Python 3.14 stdlib: `hashlib.pbkdf2_hmac` + `secrets.token_bytes` + `hmac.compare_digest`. Passwords are never stored or echoed in plain text.
- **In-memory `USERS` store** for registered accounts.
- **`POST /v1/auth/register`**
  - Validates `username` (2–32 chars, lowercase, digits, `_`, `-`)
  - Validates `password` (≥ 8 chars)
  - Validates `role` (`dm` or `player`)
  - Returns `409` for duplicate usernames
  - Returns `200` with `{"username": "...", "role": "..."}`
- **`POST /v1/auth/login`**
  - Returns `200` with `{"username": "...", "token": "session-<username>"}` on correct credentials
  - Returns `401` for bad credentials
- All existing endpoints (health, dice, checks, encounters, initiative, characters, combat sessions) are preserved untouched.

**Verified behavior:**
- Health, register, duplicate-username conflict, login, wrong-password rejection, invalid username/role rejection, and a legacy dice endpoint all returned the expected statuses.

**`./run.sh` is ready** — it starts the server in the foreground on `127.0.0.1:$PORT` using `python3 server.py`. No packages were added or changed.
