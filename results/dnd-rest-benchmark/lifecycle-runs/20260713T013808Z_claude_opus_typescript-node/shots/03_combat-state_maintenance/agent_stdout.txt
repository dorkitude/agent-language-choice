## Summary

Added **Maintenance Stage 2: Stateful Combat** to the TypeScript/Node stdlib server, preserving all existing endpoints:

- **`POST /v1/combat/sessions`** — creates an in-memory session with client-supplied `id` (duplicate → `400`), computes initiative (`roll + dex`), sorts by score desc → dex desc → name asc, starts at `round 1` / `turn_index 0`.
- **`POST /v1/combat/sessions/{id}/conditions`** — attaches a condition to a named combatant (unknown session → `404`, unknown target / non-positive duration → `400`).
- **`POST /v1/combat/sessions/{id}/advance`** — advances `turn_index`, increments `round` on wrap, decrements only the now-active combatant's conditions, and removes any that hit `0`.
