## What I added

I extended `src/server.ts` with SQLite-backed compendium APIs, preserving every existing endpoint:

**New endpoints:**
- `POST /v1/compendium/monsters` → `201` with `{slug, name, cr, armor_class, hit_points}` (tags omitted from create response, per spec)
- `GET /v1/compendium/monsters/:slug` → `200` full record including `tags`
- `POST /v1/compendium/items` → `201`
- `GET /v1/compendium/items/:slug` → `200`

**Behavior matches the contract:**
- `400` malformed input (bad/missing fields, non-integer stats, non-string tags)
- `404` unknown slug
- `409` duplicate slug
- Two new SQLite tables (`monsters`, `items`) with in-memory mirrors, wired into `rehydrate()` and `resetStorage()` — verified data survives a restart 🔁

**Preserved:** all prior routes (dice, checks, encounters, initiative, characters, auth, combat, storage) still respond correctly, tsc compiles clean, and `run.sh` is unchanged — it builds then starts the server in the foreground on `127.0.0.1:$PORT`. 🚀

I cleaned up the temporary `game.db` from my test runs so the workspace starts fresh for the evaluator.
