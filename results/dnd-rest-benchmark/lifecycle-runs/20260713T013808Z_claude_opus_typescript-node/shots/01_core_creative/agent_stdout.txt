✨ Done, Chef! The core D&D REST engine is implemented and verified.

**What I built:**
- `src/server.ts` — a single-file server using only Node's built-in `node:http` (no frameworks), TypeScript 7.0.2 + Node 26.4.0 as pinned.
- All 5 endpoints match the spec byte-for-byte on the sample payloads:
  - `GET /health` → `{"ok":true}`
  - `POST /v1/dice/stats` — regex-parsed `<count>d<sides>[±mod]`, 400 on invalid
  - `POST /v1/checks/ability` — total/success/margin
  - `POST /v1/encounters/adjusted-xp` — CR→XP table, count multipliers, summed level-3 thresholds, difficulty ladder
  - `POST /v1/initiative/order` — sort by score desc, then dex desc, then name asc

**Small config fix:** TS 7.0.2 required an explicit `rootDir` and `types: ["node"]` in `tsconfig.json` to compile cleanly.

**`run.sh`** was kept as-is (`tsc && node dist/server.js`) — it compiles then starts the server in the foreground on `127.0.0.1:$PORT`. Server is not left running. 🎲
