Fixed! 🎲

The bug was integer division in the `dice_stats` average calculation. For `1d20-1`, the average of a d20 is `10.5`, minus `1` gives `9.5`, but the code was dividing with `/ 2` and truncating to `9`.

### Change made

`app/controllers/dnd_controller.rb`:

```ruby
# before
average: dice_count * (sides + 1) / 2 + modifier,

# after
average: dice_count * (sides + 1) / 2.0 + modifier,
```

Using `2.0` forces floating-point division, so:
- `1d20-1` → `10.5 - 1 = 9.5`
- `2d6+3` → `7.0 + 3 = 10.0` (JSON numeric value matches `10`)

### Preserved behavior

- All other endpoints (`health`, `ability_check`, `adjusted_xp`, `initiative_order`) remain unchanged.
- `./run.sh` still starts the server in the foreground on `127.0.0.1:$PORT` via `bundle exec rackup`.
