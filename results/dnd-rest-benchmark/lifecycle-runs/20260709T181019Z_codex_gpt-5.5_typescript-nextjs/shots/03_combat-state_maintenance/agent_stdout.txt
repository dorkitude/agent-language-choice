Implemented the combat state stage under `app/v1/combat`:

- `POST /v1/combat/sessions`
- `POST /v1/combat/sessions/{id}/conditions`
- `POST /v1/combat/sessions/{id}/advance`

State is process-local in memory, initiative ordering follows the requested tie-breakers, conditions decrement only when the affected combatant becomes active, and unknown session IDs return `404`.
