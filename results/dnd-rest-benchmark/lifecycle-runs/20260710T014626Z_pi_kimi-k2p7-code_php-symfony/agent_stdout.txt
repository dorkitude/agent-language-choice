Fixed the bug, Chef! 🎲✨

**Root cause:** When a combatant's last condition expired, `advanceTurn` used `unset()` to remove the combatant key entirely, and the response formatter skipped empty arrays. The evaluator expected the combatant key to remain with an empty conditions array.

**Change made in `index.php`:**
- Replaced the `unset($session['conditions'][$activeName])` branch with `$session['conditions'][$activeName] = $updated;` so the key persists even when empty.
- Removed the `if ($formatted !== [])` guard in the response formatter so empty condition lists are still serialized.

**Verification:** Built the evaluator and ran the full `combat-state` suite against a temporary server:

```text
suite=combat-state base_url=http://127.0.0.1:49265 passed=true tests=20/20
```

The test server was started only for verification and has been stopped. `./run.sh` is ready and unchanged.
