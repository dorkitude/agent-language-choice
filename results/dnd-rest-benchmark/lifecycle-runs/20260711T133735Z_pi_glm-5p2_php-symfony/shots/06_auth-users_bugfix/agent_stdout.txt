## Summary

**Root cause:** The `registerUser` handler returned a `JsonResponse` with the default status (200), but the evaluator expects **201 Created** for successful user registration — a resource-creation endpoint.

**Fix:** A single, surgical change in `index.php` — the register success response now passes `201` as the status code:

```php
// Never echo the plain password in responses. 201 Created for new resource.
return new JsonResponse(['username' => $username, 'role' => $role], 201);
```
