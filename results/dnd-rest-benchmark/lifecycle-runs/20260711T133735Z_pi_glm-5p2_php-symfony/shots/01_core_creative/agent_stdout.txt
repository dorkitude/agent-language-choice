## What I built

A single front-controller `index.php` using **Symfony HttpFoundation 8.1.1** + **Symfony Routing 8.1.0** (no framework downgrade ‚Äî exact pinned versions), served by PHP 8.5.8's built-in server via `run.sh`.

### Design notes
- **Routing via Symfony components**: `RouteCollection` + `UrlMatcher` + `RequestContext::fromRequest()`, with `ResourceNotFoundException`‚Üí404 and `MethodNotAllowedException`‚Üí405.
- **Clean JSON numbers**: a `num()` helper collapses whole-valued floats to ints so `2.0`‚Üí`2`, `1700.0`‚Üí`1700` (matching the spec), while `3.5`/`1.5`/`2.5`/`437.5` stay as floats. PHP's `serialize_precision=-1` already does this, but the helper makes it explicit.
- **Average** = `(min+max)/2` ‚Äî exact mean (e.g. `1d6`‚Üí`3.5`, `2d6+3`‚Üí`10`).
- **Difficulty** cascades `deadly‚Üíhard‚Üímedium‚Üíeasy‚Üítrivial` against summed party thresholds.
- **Validation**: dice returns 400 on non-positive count/sides, malformed grammar, or extra tokens; encounter returns 400 on unknown CR or unsupported level (the spec's finite tables).

### Verified edge cases
- All 5 difficulty tiers (trivial/easy/medium/hard/deadly)
- Multiplier tiers 1, 1.5, 2, 2.5, 3, 4
- CR fractions `1/8`, `1/4`, `1/2` + integer CRs `0`‚Äì`5`
- Initiative tie-breaks (score, then dex, then name)
- Dice: uppercase `D`, negative modifiers, fractional averages, invalid expressions
- 404 + 405 handling, `run.sh` foreground startup on `127.0.0.1:$PORT`
