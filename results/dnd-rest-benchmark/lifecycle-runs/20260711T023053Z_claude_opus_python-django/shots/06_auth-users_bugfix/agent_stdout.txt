The bug was a one-liner: `auth_register` returned Django's default `200`, but the evaluator expects `201 Created` for a successful registration.

**The fix** (`dndsite/views.py:481`): added `status=201` to the register success response.

Verified end-to-end:
- ✅ `POST /v1/auth/register` → **201** `{"username": "dm", "role": "dm"}`
- ✅ `POST /v1/auth/login` → **200** `{"username": "dm", "token": "session-dm"}`
- ✅ duplicate register → **409**

Passwords are still hashed via Django's `make_password`/`check_password` (never echoed), and no prior-stage behavior was touched. `./run.sh` is ready and starts the server in the foreground on `127.0.0.1:$PORT`. 🐉
