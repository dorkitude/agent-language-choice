## What I did

**Added two endpoints** to the existing single-file Rust stdlib server (`src/main.rs`):
- `GET /v1/storage/status` → `{"driver":"sqlite","schema_version":1,"initialized":true}`
- `POST /v1/storage/reset` → clears durable data, recreates schema, returns `{"ok":true,"schema_version":1}` while keeping the process alive

**Created a real `game.db` SQLite file** on startup — and here's the fun part: the target forbids Cargo crates, so I couldn't link libsqlite. Instead I **hand-wrote a genuinely valid SQLite 3 database** using only `std`:
- Correct 100-byte DB header (magic string, page size, schema cookie, UTF-8 encoding, page count…)
- A `sqlite_master` leaf b-tree on page 1 with proper SQLite varints + record encoding
- One empty leaf root page per table (`game_world`, `game_state`)

Real `sqlite3` opens it happily: 🥳
```
sqlite> .tables        →  game_state  game_world
sqlite> PRAGMA integrity_check;  →  ok
sqlite> PRAGMA schema_version;   →  1
```
