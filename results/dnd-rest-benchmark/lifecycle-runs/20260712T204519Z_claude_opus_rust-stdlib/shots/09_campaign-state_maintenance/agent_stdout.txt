Added **Campaign State APIs** to the Rust stdlib server, following the existing in-memory-store-plus-SQLite-schema pattern:

- **`POST /v1/campaigns`** → creates a campaign; `409` on duplicate id
- **`POST /v1/campaigns/{id}/characters`** → adds a character; `404` if campaign unknown, `409` on duplicate character id
- **`POST /v1/campaigns/{id}/events`** → logs an event (returns `{id, kind}`); `404`/`409` as above
- **`GET /v1/campaigns/{id}/state`** → full state with characters + `log_count`; `404` if unknown

**Validation** matches the spec: `400` for malformed/missing fields, `404` for unknown campaigns, `409` for duplicate IDs. 🧙
