The core D&D REST engine is implemented in `src/main.rs` using **Rust 1.97.0 stdlib only** — no Cargo deps, no HTTP/JSON crates.

**What I built:**
- 🌐 A raw HTTP layer on `TcpListener`/`TcpStream` with proper request reading (headers + `Content-Length` body, replacing the fragile single-`read` starter).
- 🧩 A tiny hand-rolled recursive JSON parser (`Json` enum + accessors) and a JSON string serializer — serde-free.
- All five endpoints:
  - `GET /health` → `{"ok":true}`
  - `POST /v1/dice/stats` → parses `<count>d<sides>[±mod]`, validates strictly (400 on bad input), computes min/max/average
  - `POST /v1/checks/ability` → total/success/margin
  - `POST /v1/encounters/adjusted-xp` → CR→XP table, count multipliers, summed level-3 thresholds, difficulty tiering
  - `POST /v1/initiative/order` → sorts by score desc → dex desc → name asc

**Verified against the spec examples** — every response byte matches, including whole-number formatting (`average:10` vs `average:3.5`), the `deadly` sample, and 400s for invalid dice.

`./run.sh` is ready (unchanged — it compiles with `--edition=2024` and runs the server in the foreground on `127.0.0.1:$PORT`). I did **not** start the server as part of finishing. 🚀
