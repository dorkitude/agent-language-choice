I added the three combat-state endpoints to the existing Sinatra app while preserving all prior core and character-rule behavior:

- `POST /v1/combat/sessions` — creates a session, computes initiative (`roll + dex`), sorts by score desc → dex desc → name asc, and returns `round`, `turn_index`, `active`, and `order`.
- `POST /v1/combat/sessions/:id/conditions` — attaches a condition with positive `duration_rounds` to a known combatant.
- `POST /v1/combat/sessions/:id/advance` — advances `turn_index`, wraps it to increment `round`, and ticks down conditions on the newly active combatant only.

All state is kept in the in-memory `COMBAT_SESSIONS` hash. Invalid requests return `400`; unknown session IDs return `404`.

`./run.sh` is unchanged and still starts the server on `127.0.0.1:$PORT` in the foreground. I verified `ruby -c app.rb` and `bundle check` are happy. I didn't start the server per your instructions.
