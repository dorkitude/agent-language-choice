Hey Chef! 🎲✨ The core D&D REST engine is ready to roll.

**What I built:**
- `app.rb` — implements all five endpoints:
  - `GET /health`
  - `POST /v1/dice/stats`
  - `POST /v1/checks/ability`
  - `POST /v1/encounters/adjusted-xp`
  - `POST /v1/initiative/order`
- `Gemfile` — added the missing `rackup` gem at the pinned version `2.3.1` so Sinatra 4 + Rack 3 can actually start with Puma.
- `run.sh` — kept as-is; it starts the server on `127.0.0.1:$PORT` in the foreground.

**Root cause found:** The starter Gemfile only listed `sinatra`, `rack`, and `puma`, but Sinatra 4 on Rack 3 requires the `rackup` gem to bootstrap a handler. Without it, the server errored out with “Sinatra could not start, the required gems weren’t found!” I added `rackup` at the exact pinned version, and now `./run.sh` boots cleanly with Puma 8.0.2.

I verified the endpoints with both `Rack::MockRequest` and a live curl against a backgrounded `./run.sh` (and killed the process afterward). All responses match the spec.
