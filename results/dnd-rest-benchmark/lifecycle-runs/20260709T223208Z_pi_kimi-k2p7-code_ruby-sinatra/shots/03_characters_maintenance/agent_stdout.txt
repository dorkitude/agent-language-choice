I added the three new character endpoints to `app.rb` while preserving every existing route:

- `POST /v1/characters/ability-modifier`
- `POST /v1/characters/proficiency`
- `POST /v1/characters/derived-stats`

Key implementation notes:

- Ability modifiers use `(score - 10).div(2)` so negative halves floor correctly (e.g., `9 → -1`).
- Proficiency bonus follows the standard tier table.
- `derived-stats` validates the level, all six ability scores, and the armor block, then computes `hp_max` and `armor_class` exactly as specified.
- Invalid requests still return `400` with a JSON error body.

`./run.sh` is already in place and will start the server in the foreground on `127.0.0.1:$PORT` using the pinned Ruby/Sinatra/Rack/Puma versions.
