Added the three DM Tools endpoints to `index.php`:
- `POST /v1/dm/encounter-builder` — reuses `LEVEL_THRESHOLDS`/`countMultiplier`/`MONSTER_XP`, looks up monster CR via compendium by slug, returns difficulty + deterministic recommendation.
- `POST /v1/dm/loot-parcel` — deterministic tier-1 loot; non-1 tiers rejected with 400.
- `POST /v1/dm/session-recap` — pulls campaign events (new `listCampaignEvents` helper ordered by `rowid`), events with `kind: "open_thread"` populate `open_threads`, others feed `summary`.

Verified against a live server: all three example payloads reproduce the spec's exact responses, invalid/missing-campaign requests return 404, malformed bodies return 400, and prior endpoints (`/v1/encounters/adjusted-xp`, `/v1/campaigns/{id}/state`, etc.) still behave as before.
