Done. Added SQLite-style compendium storage for monsters and items (in-process `HashMap` stores, mirroring the existing `users`/`combat_sessions` pattern) with:

- `POST /v1/compendium/monsters` — creates, returns `409` on duplicate slug, `400` on invalid input
- `GET /v1/compendium/monsters/:slug` — returns `404` if missing
- `POST /v1/compendium/items` — same conflict/validation semantics
- `GET /v1/compendium/items/:slug`

Both stores are cleared by `/v1/storage/reset` alongside existing state. Verified via manual `curl` testing that all new endpoints and every pre-existing endpoint (health, auth/register) still work correctly. `./run.sh` compiles `src/main.rs` and runs the server in the foreground on `127.0.0.1:$PORT`, unchanged.
