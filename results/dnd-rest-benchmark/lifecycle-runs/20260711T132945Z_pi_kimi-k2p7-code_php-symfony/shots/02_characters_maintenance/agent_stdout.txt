Added the three new character endpoints to the existing Symfony-routed PHP API:

- `POST /v1/characters/ability-modifier` — D&D ability modifier with proper `floor()` for negative values.
- `POST /v1/characters/proficiency` — proficiency bonus by level.
- `POST /v1/characters/derived-stats` — HP max, AC, modifiers, and proficiency bonus.

All existing endpoints (`/health`, `/v1/dice/stats`, `/v1/checks/ability`, `/v1/encounters/adjusted-xp`, `/v1/initiative/order`) are preserved. Invalid requests return `400`. `./run.sh` is unchanged and ready: it starts the PHP dev server in the foreground on `127.0.0.1:$PORT`.

I verified the new endpoints plus edge cases against the running server, and no server is left running now.
