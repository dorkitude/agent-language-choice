I added the missing `/v1/combat/sessions` endpoints to the existing Vite middleware in `vite.config.ts`:

- **POST `/v1/combat/sessions`** — creates a session, computes initiative (`roll + dex`), and sorts by score desc → dex desc → name asc.
- **POST `/v1/combat/sessions/{id}/conditions`** — attaches a condition to a combatant with a positive duration.
- **POST `/v1/combat/sessions/{id}/advance`** — advances the turn, wraps the round, and decrements/removes conditions on the newly active combatant.

State is kept in an in-memory `Map` inside `configureServer`, so it lives for the server process. All prior endpoints (`/health`, dice, checks, encounters, initiative, character rules) are preserved.
