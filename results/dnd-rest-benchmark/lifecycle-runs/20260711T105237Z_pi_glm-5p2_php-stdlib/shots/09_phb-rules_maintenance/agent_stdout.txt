## Summary

Added three deterministic, stateless PHB-rule endpoints to `index.php` (no Composer packages, pure PHP stdlib — matches the target guidance):

### `POST /v1/phb/spell-slots`
- Validates `class` (string, must be `wizard`) and `level` (int 1–20).
- Looks up the full PHB wizard spell-slot table (levels 1–20), so wizard level 5 returns exactly `{"1": 4, "2": 3, "3": 2}`.
- Response: `{"class":"wizard","level":5,"slots":{"1":4,"2":3,"3":2}}` ✓

### `POST /v1/phb/rests/long`
- Validates all five integer fields (`level`, `hp_current`, `hp_max`, `hit_dice_spent`, `exhaustion_level`).
- HP → max; hit dice recover `max(1, floor(level/2))` (so 5 → 2, 3→1 spent); exhaustion −1 floored at 0.
- Response: `{"hp_current":35,"hit_dice_spent":1,"exhaustion_level":0}` ✓

### `POST /v1/phb/equipment-load`
- Validates `strength` (int ≥1) and `weight` (int ≥0).
- `capacity = strength * 15`; `encumbered = weight > capacity`.
- Response: `{"capacity":180,"weight":181,"encumbered":true}` ✓

### Verification
- `php -l index.php` → no syntax errors.
- All three spec examples return exact expected JSON with HTTP 200.
- Invalid requests (bad class, missing field) return 400.
- Regression: `/health`, `/v1/dice/stats`, `/v1/characters/proficiency`, `/v1/storage/status`, `/v1/auth/register` all still behave correctly.

`run.sh` is unchanged and already starts the built-in PHP server in the foreground on `127.0.0.1:$PORT` — ready to go. 🚀
