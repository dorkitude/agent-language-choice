## Summary

Created `index.php` — a pure PHP stdlib router (no Composer packages) for the built-in PHP 8.5.8 server. `run.sh` was already present and correct (starts `php -S 127.0.0.1:$PORT index.php` in the foreground).

**Endpoints implemented & tested:**

| Endpoint | Status | Behavior |
|---|---|---|
| `GET /health` | ✅ | `{"ok":true}` |
| `POST /v1/dice/stats` | ✅ | Parses `<count>d<sides>[±<mod>]`, returns dice_count/sides/modifier/min/max/average; **400** on invalid expr (regex + positivity check) |
| `POST /v1/checks/ability` | ✅ | `total=roll+modifier`, `success=total>=dc`, `margin=total-dc` |
| `POST /v1/encounters/adjusted-xp` | ✅ | CR→XP table, count multipliers (1/1.5/2/2.5/3/4), level-3 thresholds summed across party, difficulty = highest threshold reached |
| `POST /v1/initiative/order` | ✅ | `score=roll+dex`, sorted by score desc → dex desc → name asc (stable) |

**Key correctness details handled:**
- **Int/float JSON encoding** — PHP's natural arithmetic produces clean output: `average: 10` (int) vs `3.5` (float), `multiplier: 2` vs `1.5`, `adjusted_xp: 1700` vs `112.5` (verified `json_encode` drops `.0` on whole floats).
- **400s** for malformed JSON, missing fields, non-positive count/sides, unsupported CR, unsupported level.
- **Tie-breaking** in initiative verified for all three levels (score, then dex, then name).
- **Fractional averages/adjusted-XP** kept mathematically exact (no silent rounding).

All sample responses match the spec byte-for-byte (modulo insignificant JSON whitespace, which any parser normalizes). No servers left running. 🐉
