## Summary

I added the **auth-users** stage to the PHP stdlib D&D REST API while preserving all prior-stage behavior. Here's what I did:

### Changes to `index.php`

1. **User storage helpers** (`users_state_file`, `load_users`, `save_users`) ‚Äî file-backed JSON storage (`.users.json`), same pattern as the existing combat-session storage, since PHP's built-in server re-executes per request and has no in-memory state.

2. **Password helpers** (`hash_password`, `verify_password`) ‚Äî using PHP's standard `password_hash`/`password_verify` (bcrypt `$2y$` by default). Isolated behind small helpers so a production hash could drop in cleanly. Plain passwords are never stored or echoed.

3. **`handle_register()`** ‚Äî validates:
   - username: 2‚Äì32 chars, `[a-z0-9_-]` ‚Üí else `400`
   - password: ‚â•8 chars ‚Üí else `400`
   - role: `dm` or `player` ‚Üí else `400`
   - duplicate username ‚Üí `409`
   - success ‚Üí **`201`** with `{"username", "role"}` (no password echoed)
