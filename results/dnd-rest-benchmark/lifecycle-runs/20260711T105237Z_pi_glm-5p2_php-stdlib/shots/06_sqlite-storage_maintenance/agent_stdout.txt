## Changes

**`db.php` (new)** — SQLite storage module using PDO + `pdo_sqlite` (no Composer packages):
- Schema v1 tables: `schema_meta`, `combat_sessions` (JSON blob per id), `users` (normalized)
- `db()` opens a fresh connection per request and idempotently ensures the schema (the built-in server re-executes per request, so this guarantees `initialized: true`)
- `db_reset()` drops + recreates the data tables
- Drop-in `load/save_combat_sessions` and `load/save_users` replacing the old JSON-file helpers → **handlers untouched**
- Runnable directly (`php db.php`) for startup pre-init

**`index.php`** — replaced the JSON-file storage helpers with `require __DIR__ . '/db.php'`, and added two endpoints:
- `GET /v1/storage/status` → `{"driver":"sqlite","schema_version":1,"initialized":true}`
- `POST /v1/storage/reset` → `{"ok":true,"schema_version":1}` (drops + recreates schema, clears all durable data)

**`run.sh`** — wipes `game.db` on startup (preserves the prior per-process-lifetime semantics), pre-initializes the schema, then starts the built-in server on `127.0.0.1:$PORT`.
