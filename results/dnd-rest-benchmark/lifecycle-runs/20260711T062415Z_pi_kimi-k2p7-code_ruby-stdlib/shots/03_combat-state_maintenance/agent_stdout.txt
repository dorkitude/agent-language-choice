**What changed in `server.rb`:**

- `POST /v1/combat/sessions` — creates a session with client-supplied `id`, computes initiative (`roll + dex`), sorts by score desc → dex desc → name asc, and starts at `round: 1`, `turn_index: 0`.
- `POST /v1/combat/sessions/{id}/conditions` — attaches a condition with positive `duration_rounds` to a named combatant.
- `POST /v1/combat/sessions/{id}/advance` — advances `turn_index`, wraps to the next round, and decrements/removes conditions on the newly active combatant only.

All state lives in an in-memory `SESSIONS` hash, and error handling returns `400` for malformed input and `404` for unknown session IDs, matching the existing style. I also verified the full round wrap-around and condition decay behavior.

`./run.sh` was already correct and runs the server in the foreground on `127.0.0.1:$PORT`. I tested locally and killed the temporary processes, so nothing is lingering. 🧙‍♂️🗡️
