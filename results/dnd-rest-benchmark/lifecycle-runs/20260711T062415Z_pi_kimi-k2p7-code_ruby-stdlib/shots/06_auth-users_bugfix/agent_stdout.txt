Fixed! 🐛✨

**Root cause:** The `POST /v1/auth/register` success handler was returning HTTP 200, but the evaluator expects HTTP 201 for a successful registration.

**Changes made in `server.rb`:**
- Added `201 => 'Created'` to `STATUS_TEXT`.
- Changed the successful `handle_register` response from `json_response(200, ...)` to `json_response(201, ...)`.

`./run.sh` is unchanged and ready to start the server in the foreground on `127.0.0.1:${PORT}` when you run it, Chef. 🎲
