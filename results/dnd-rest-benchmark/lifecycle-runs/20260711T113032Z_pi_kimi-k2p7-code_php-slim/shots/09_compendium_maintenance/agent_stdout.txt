I added the SQLite-backed Monster & Item compendium to the existing Slim app while keeping all prior endpoints intact.

**What changed in `index.php`:**
- Added `compendium_monsters` and `compendium_items` tables in `initDb()`, and drop them in `resetDb()`.
- Added helper functions: `createMonster`, `findMonsterBySlug`, `createItem`, `findItemBySlug`.
- Added routes:
  - `POST /v1/compendium/monsters` → creates a monster (returns 201; omits `tags` as shown).
  - `GET /v1/compendium/monsters/{slug}` → reads a monster (includes `tags`).
  - `POST /v1/compendium/items` → creates an item (returns 201).
  - `GET /v1/compendium/items/{slug}` → reads an item.
- Malformed requests return `400`, missing records return `404`, and duplicate slugs return `409`.
- Kept `schema_version` at `1` so the previous storage status contract still passes.

`run.sh` is already ready and starts the server in the foreground on `127.0.0.1:$PORT`. I also ran `php -l index.php` and it’s syntax-clean.
