Chef, the combat stage is ready! 🎲⚔️

What I added:

- **`POST /v1/combat/sessions`** — creates a client-supplied combat session, computes initiative (`roll + dex`), and sorts by score desc → dex desc → name asc.
- **`POST /v1/combat/sessions/{id}/conditions`** — attaches a condition with positive duration to a named combatant.
- **`POST /v1/combat/sessions/{id}/advance`** — advances turn, wraps the round, and decrements conditions on the newly active combatant, removing them at 0.

All prior endpoints (`/health`, dice, checks, encounters, initiative, character rules) are preserved exactly.

State detail: PHP’s built-in dev server resets globals between requests, so I keep the combat state in a tiny `.combat_state.json` file in the workspace. `run.sh` clears it on every start, so the state is effectively tied to the server process lifetime.

`./run.sh` is ready and listens on `127.0.0.1:$PORT`. I’m not starting the server now.
