I added the three new character-rule endpoints to `index.php` while preserving all existing core endpoints:

- `POST /v1/characters/ability-modifier`
- `POST /v1/characters/proficiency`
- `POST /v1/characters/derived-stats`

Key implementation notes:
- Used `floor((score - 10) / 2)` for ability modifiers so negative halves floor correctly (e.g., score 9 → −1).
- Added a shared `proficiencyBonus()` helper and a `validateScore()` helper.
- Derived stats validates `level`, all six abilities, and the `armor` object before computing `hp_max`, `armor_class`, and modifiers.
- All success responses are JSON; invalid requests return `400`.

`./run.sh` is unchanged and still starts the PHP built-in server on `127.0.0.1:$PORT` in the foreground.
