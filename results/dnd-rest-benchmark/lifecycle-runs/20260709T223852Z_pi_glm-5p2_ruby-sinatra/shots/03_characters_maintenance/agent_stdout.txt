## What I added

Three new endpoints in `app.rb`, plus two top-level helpers (`ability_modifier`, `proficiency_bonus`):

**`POST /v1/characters/ability-modifier`** — validates `score` is an integer 1–30, returns `{"score":..., "modifier":...}`. Ruby's `Integer#/` floors toward −∞, so score 9 → `(9-10)/2 = -1` ✓

**`POST /v1/characters/proficiency`** — validates `level` is an integer 1–20, returns `{"level":..., "proficiency_bonus":...}` via a `case` matching the tier table (1–4→2, 5–8→3, … 17–20→6).

**`POST /v1/characters/derived-stats`** — validates `level` (1–20), all six abilities (str/dex/con/int/wis/cha, each int 1–30), and `armor` (`base` + `dex_cap` ints, `shield` optional → `shield_bonus` 2 only when `armor.shield == true`). Computes:
- `hp_max = level * (6 + con_modifier)`
- `armor_class = base + min(dex_modifier, dex_cap) + shield_bonus`
