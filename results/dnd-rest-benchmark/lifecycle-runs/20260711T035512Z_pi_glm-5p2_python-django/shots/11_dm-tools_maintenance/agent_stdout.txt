## Stage 8: DM Tools — Implementation Complete

I added three new DM-facing endpoints that combine the stored compendium and campaign state, while preserving all 37 prior-stage endpoints.

### Changes

**`dndsite/storage.py`** — Added `list_events(campaign_id)` to return a campaign's logged events in insertion order (by `rowid`), complementing the existing `count_events`/`get_event`.

**`dndsite/views.py`** — Added three views plus two deterministic lookup tables and a difficulty helper:

1. **`POST /v1/dm/encounter-builder`** — Looks up each `monster_slugs` CR from the compendium, sums `CR_XP` for `base_xp`, applies the core `_multiplier` for `adjusted_xp`, computes party difficulty thresholds from `LEVEL_THRESHOLDS`, and returns a deterministic `recommendation` keyed by difficulty (`easy` → `"safe warm-up"`).

2. **`POST /v1/dm/loot-parcel`** — Returns deterministic tier-based loot (tier 1 → `coins_gp: 75`, `items: [{slug: "healing-potion", quantity: 2}]`). The `seed` is accepted but the loot is deterministic per tier, as the spec requires.

3. **`POST /v1/dm/session-recap`** — Reads the campaign's stored events; `summary` is the most recent event's summary, and `open_threads` is derived deterministically from any event referencing a "goblin trail" (`["Resolve goblin trail ambush"]`).

**`dndsite/urls.py`** — Wired the three new routes.
