## What I built

A minimal Django 6.0.7 REST engine implementing the **core** stage endpoints, wired into the seeded `dndsite` project:

**`dndsite/views.py`** (new) — five view functions:
- `GET /health` → `{"ok": true}`
- `POST /v1/dice/stats` — regex-parsed `<count>d<sides>[±<mod>]`, positive count/sides enforced, 400 on invalid. `average = (min+max)/2` (int when whole, e.g. `10`; float when not, e.g. `9.5`).
- `POST /v1/checks/ability` — `total = roll+modifier`, `success = total >= dc`, `margin = total-dc`.
- `POST /v1/encounters/adjusted-xp` — CR→XP table, monster-count multipliers (1 / 1.5 / 2 / 2.5 / 3 / 4), summed party thresholds (level 3), highest-reached difficulty (`trivial`→`deadly`).
- `POST /v1/initiative/order` — `score = roll+dex`, sorted by score desc → dex desc → name asc.

**`dndsite/urls.py`** — routes wired to the views (no trailing slashes, matching the spec).

**`run.sh`** — kept the `PYTHONPATH=.deps` + `runserver --noreload` pattern, added `PORT="${PORT:-8000}"` fallback so `set -u` never trips; listens on `127.0.0.1:$PORT` in the foreground.
