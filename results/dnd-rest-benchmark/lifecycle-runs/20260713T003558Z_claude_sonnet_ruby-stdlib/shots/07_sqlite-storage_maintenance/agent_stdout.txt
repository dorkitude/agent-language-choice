SQLite-backed storage stage is done, Chef! 🎲✨

I added `game.db` (SQLite via the system `sqlite3` gem, since stdlib alone has no SQLite driver) with `users` and `combat_sessions` tables, initialized on startup via `init_db!`. Users and combat sessions now write-through to the DB while keeping the existing in-memory hashes for request-path logic, so all prior endpoints behave identically. Added `GET /v1/storage/status` and `POST /v1/storage/reset` per spec — verified with curl that health, register/login, combat session creation, and reset all work correctly, and that reset actually clears data (login fails after reset). `./run.sh` still runs `ruby server.rb` in the foreground on `127.0.0.1:$PORT`.
