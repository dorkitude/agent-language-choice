I added the **stateful combat** endpoints to `server.py` while keeping all existing core/character endpoints intact:

- `POST /v1/combat/sessions` — creates a session with deterministic initiative ordering
- `POST /v1/combat/sessions/{id}/conditions` — attaches a timed condition to a combatant
- `POST /v1/combat/sessions/{id}/advance` — advances the turn, increments the round on wrap, and ticks down conditions on the newly active combatant

All state is in-memory, and invalid inputs return `400` while unknown session IDs return `404`.

`./run.sh` is already in place and starts the server in the foreground on `127.0.0.1:${PORT}` via `python3 server.py`. I verified the new endpoints and the existing suite with curl tests.
