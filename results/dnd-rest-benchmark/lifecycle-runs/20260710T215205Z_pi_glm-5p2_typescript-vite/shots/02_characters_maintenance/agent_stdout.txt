## What changed

I edited **`src/dndPlugin.ts`** (the existing Vite dev-server middleware plugin) to add three new character-rule endpoints, leaving all prior core endpoints untouched:

### New endpoints
1. **`POST /v1/characters/ability-modifier`** — validates `score` is an integer 1–30, returns `{score, modifier}` where `modifier = Math.floor((score - 10) / 2)`. `Math.floor` correctly floors negative halves (score 9 → -1). ✅
2. **`POST /v1/characters/proficiency`** — validates `level` is an integer 1–20, returns `{level, proficiency_bonus}` computed as `Math.floor((level - 1) / 4) + 2` (yields 2/3/4/5/6 across the four-level tiers). ✅
3. **`POST /v1/characters/derived-stats`** — validates `level`, all six abilities (1–30), and `armor` (`base` int, `shield` optional bool, `dex_cap` int); computes `hp_max = level * (6 + con_mod)`, `armor_class = base + min(dex_mod, dex_cap) + (shield ? 2 : 0)`, and all six modifiers. ✅

Invalid inputs return `400` with a JSON error body.
