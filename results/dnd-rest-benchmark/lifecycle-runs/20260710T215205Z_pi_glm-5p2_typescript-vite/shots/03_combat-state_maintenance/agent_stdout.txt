### What I added (in `src/dndPlugin.ts`)

A new in-memory combat section with three endpoints, all routed through the existing Vite dev-server middleware plugin (no server replacement):

1. **`POST /v1/combat/sessions`** вЂ” creates a session from a client-supplied `id` + combatants. Computes `score = roll + dex`, sorts by score desc в†’ dex desc в†’ name asc, starts `round=1`, `turn_index=0`, returns `active` + `order`.

2. **`POST /v1/combat/sessions/{id}/conditions`** вЂ” attaches a condition (arbitrary string) with a positive-integer `duration_rounds` to a named combatant. Validates the target exists in the session.

3. **`POST /v1/combat/sessions/{id}/advance`** вЂ” advances `turn_index`, increments `round` on wrap, decrements conditions on the *newly active* combatant only, and removes conditions that hit `0`.
